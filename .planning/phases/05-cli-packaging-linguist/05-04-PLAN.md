Implement `voss init` with safe project scaffolding, package-bundled templates, a parseable hello-world Voss program, and the required project `.gitattributes` override.
</objective>

<threat_model>
Threats:
- T-05-04-A: `init` overwrites existing user files or writes outside the target directory.
- T-05-04-B: Templates work only from the source tree and are missing after package installation.
- T-05-04-C: Scaffold includes live provider credentials, network-dependent defaults, or unparseable Voss code.
- T-05-04-D: Scaffold Linguist override is incomplete or differs from TOOL-02's exact required line.

Mitigations:
- Tests cover non-empty destination refusal, `--force`, and project-local path containment.
- Template reads use `importlib.resources` and package data is updated in `pyproject.toml`.
- Scaffold Voss source parses through the real parser and contains no API-key placeholders.
- Tests assert the exact `.gitattributes` line.
</threat_model>
