<objective>
Implement the write/execute CLI commands: `voss compile` for generated Python output and `voss run` for subprocess-backed execution through the current Python interpreter.
</objective>

<threat_model>
Threats:
- T-05-03-A: `compile` ignores analyzer errors and writes runnable Python anyway.
- T-05-03-B: Output paths overwrite unrelated files or escape the intended project scope by default.
- T-05-03-C: `run` executes generated code in-process and leaks imports, globals, `sys.argv`, or provider state into the CLI process.
- T-05-03-D: Default tests accidentally call live providers, network APIs, or model downloads.

Mitigations:
- Tests assert analyzer errors block writes and warnings do not block by default.
- Default output is `SOURCE.py` beside the source; explicit `--output` writes atomically after successful generation.
- `run` uses `subprocess.run([sys.executable, generated_path, *args])` and returns the subprocess exit code.
- Tests monkeypatch public compiler APIs or use hermetic generated scripts; no live provider credentials are required.
</threat_model>
