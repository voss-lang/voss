<objective>
Implement the read-only CLI commands: `voss ast` for deterministic AST JSON and `voss check` for analyzer diagnostics without compile artifacts, code execution, provider calls, or `.voss-cache` writes.
</objective>

<threat_model>
Threats:
- T-05-02-A: `check` creates semantic indexes or `.voss-cache` artifacts while users expect lint-only behavior.
- T-05-02-B: `ast` or `check` executes generated/user Python or imports user `use` targets.
- T-05-02-C: CLI exit codes treat warnings inconsistently and make CI behavior unpredictable.

Mitigations:
- Tests assert no generated `.py` files or `.voss-cache` entries are created.
- `ast` calls only `parse` and `to_dict`; `check` calls only `parse` and `analyze(..., emit_indexes=False)`.
- Exit policy is explicit: errors fail, warnings pass unless `--warnings-as-errors` is set.
</threat_model>
