## Spacing Scale

F3 inherits the full A1 spacing scale without modification. Budget-HUD-specific
measurements are declared as named structural constants below.

| Token | Value | F3 Usage |
|-------|-------|---------|
| xs | 4px | Gap between cost text and bar track; gap between bar and `⋯` menu; popover row label/value gap |
| sm | 8px | Horizontal padding inside BudgetBar segment; popover internal padding |
| md | 16px | Popover horizontal padding from left edge to content |
| lg | 24px | Not used in budget HUD chrome |
| xl | 32px | Not used in budget HUD chrome |

**Exceptions:** none. BudgetBar is constrained to the 22px PaneHeader strip — all internal spacing follows xs/sm.

### Named Structural Constants for F3 Budget HUD Chrome

| Constant | Value | Rationale |
|----------|-------|-----------|
| PaneHeader height | **22px** | A3-05 Variant B fixed height. Budget bar must fit within this. |
| BudgetBar track height | **4px** | 1×4 grid unit. Sits vertically centered in the 22px header. Visible enough without competing with text. |
| BudgetBar track width | **48px** | 12×4 grid. Wide enough to show progress resolution; narrow enough for cramped headers. |
| BudgetBar fill min-width | **2px** | Prevents fully-empty bar from rendering as invisible. |
| BudgetBar segment total width | **~80px** | Cost text (max ~40px) + 4px gap + 48px track + 4px gap before `⋯`. Flex-shrink: 0. |
| Cost text max-width | **44px** | Accommodates `$99.9999` (6 chars at 11px mono). Overflow clips. |
| Budget popover z-index | **20** | Above PaneHeader (z:0), below command palette / settings overlays (z:50). Matches A10 popover z-index. |
| Budget popover width | **220px** | 4 fields × label+value layout. Narrower than A10's center/right popovers (detail card not navigation). |
| Budget popover max-height | **auto** | Fixed 5 rows — no scroll needed. Content-driven height. |
| Popover row height | **24px** | 6×4 grid — label + value on same line, mono font. Matches A10 pane detail card row height. |
| Popover header height | **24px** | Matches A10 popover heading row height. |
| Bar transition duration | **150ms** | D-13. CSS `transition: width 150ms ease-out`. Kill switch: `prefers-reduced-motion`. |

**Border radius:** 0px on all F3 chrome — Variant B absolute rule. No exceptions.

## Typography

Budget HUD is terminal-density chrome. Mono font throughout. Single size tier.

| Role | Size | Weight | Line Height | Font | F3 Usage |
|------|------|--------|-------------|------|---------|
| Cost label | 11px | 400 | 1.0 | `--font-mono` | `$0.08` inline in PaneHeader — matches all PaneHeader text |
| Popover heading | 11px | 500 | 1.0 | `--font-mono` | "Budget Detail" heading row |
| Popover label | 11px | 400 | 1.0 | `--font-mono` | Field labels: `tokens:`, `limit:`, `model:`, `turns:`, `cost:` |
| Popover value | 12px | 400 | 1.4 | `--font-mono` | Field values; steps up one size for readability vs. label column |

**Size rationale:** 11px matches PaneHeader's established `font-size: 11px` (confirmed in `PaneHeader.tsx`). Popover values step up to 12px — matching A10's pane detail card `value` column. Two sizes maximum, same as A10.

## Color

F3 uses A1 tokens verbatim. Budget-HUD-specific usage extensions below are additive only.

### Background — F3 Budget HUD Usage

| Token | Hex | F3 Usage |
|-------|-----|---------|
| `--bg-2` | `#171a23` | Budget bar track (empty fill); matches PaneHeader focused background (`bg-bg-2`). |
| `--bg-3` | `#1f232e` | Budget popover card background; popover header row. |

### Foreground — F3 Budget HUD Usage

| Token | Hex | F3 Usage |
|-------|-----|---------|
| `--fg-2` | `#6a7080` | Cost text (`$0.08`) in PaneHeader. Subdued — secondary data, not primary cwd/process text. |
| `--fg-1` | `#aab0c0` | Popover field values (tokens used, model name, cost, iterations). |
| `--fg-3` | `#444a5a` | Popover field label text; separator glyphs; empty/loading placeholder text. |

### Accent — F3 Budget Bar Fill (3-Tier, D-08)

| Token | Hex | Budget Threshold | Behavior |
|-------|-----|-----------------|---------|
| `--accent-green` | `#6fd28f` | 0–70% | Normal. Bar fill color at low-to-mid consumption. |
| `--accent-amber` | `#e8b86c` | 70–90% | Warning. Bar transitions to amber at 70% of token limit. |
| `--accent-red` | `#e87b7b` | 90–100% | Error. Bar transitions to red at 90% of token limit. |

**No-limit state (D-07):** When `token_limit` is absent, no bar renders. Only cost text shows. No accent color applied — `--fg-2` for the text only.

**Cost text color (always `--fg-2`):** Cost text does not change color by threshold. Only the bar fill transitions. This prevents two simultaneous color-state signals competing in the header.

**60/30/10 split:** `--bg-1`/`--bg-2` = 60% (PaneHeader surface + bar track), `--bg-3` = 30% (popover body + header), accent fill = 10% (bar fill only).

## Layout Contract

### PaneHeader Structure — Before F3

```
┌──────────────────────────────────────────────────────────────┐  22px
│ ● │ 2 │ ~/projects/voss │ zsh │ cargo build          [spacer] ⋯│
└──────────────────────────────────────────────────────────────┘
```

### PaneHeader Structure — After F3 (agent panes only)

```
┌──────────────────────────────────────────────────────────────┐  22px
│ ● │ 2 │ ~/projects/voss │ zsh │ cargo build  [spacer] $0.08 ▓▓░░░░ ⋯│
│                                              ←── BudgetBar ──→       │
└──────────────────────────────────────────────────────────────┘
```

The BudgetBar segment is inserted between the `[spacer]` flex expander and the `⋯` menu button. It is `flex-shrink: 0` — it never compresses. The spacer (`flex: 1`) absorbs all available space to the left.

**Shell panes:** BudgetBar segment is absent entirely. No placeholder space. No empty cost text.

**Agent pane, no budget received yet:** BudgetBar absent. Nothing rendered until the first `BudgetUpdate` OSC event arrives (D-11: blank between mount and first emission is acceptable).

**Agent pane, `token_limit` absent (D-07):** Render cost text only. No bar track, no bar fill.

```
│ ... [spacer] $0.08 ⋯ │
```

**Agent pane, `token_limit` present (D-06):**

```
│ ... [spacer] $0.08 [track: ▓▓▓▓░░░░░░░░] ⋯ │
               ↑ --fg-2  ↑ --bg-2 track, --accent fill
```

### BudgetBar Internal Layout

```
┌──────────────────────────────────────────────────┐
│  $0.08  [  ▓▓▓▓░░░░░░░░  ]                       │
│  ←4px→   ←──── 48px ─────→                       │
│  cost     track: --bg-2                           │
│  --fg-2   fill: --accent-* (tier-dependent)       │
└──────────────────────────────────────────────────┘
  Total segment: flex-shrink: 0; 4px right padding before ⋯
```

- Cost text: `max-width: 44px`, `white-space: nowrap`, `overflow: hidden`
- 4px gap between cost text and bar track
- Bar track: `width: 48px`, `height: 4px`, `background: --bg-2`, vertically centered
- Bar fill: `height: 4px`, `width: {pct}%`, `background: var(--accent-*)`, `transition: width 150ms ease-out`
- 4px gap between bar track and `⋯` button
- Entire BudgetBar segment is a `<button>` element (click opens popover, D-09)

### Budget Popover Positioning

Popover opens **downward** from the BudgetBar anchor, aligned to the right edge of the BudgetBar segment. If popover would overflow viewport right edge, shift left until fully visible.

```
                          ┌──────────────────────────┐
                          │ Budget Detail       11px  │  ← popover opens below header
                          ├──────────────────────────┤
                          │ tokens:  12,450 / 20,000  │
                          │ limit:   20,000 tokens     │
                          │ model:   claude-sonnet-4   │
                          │ turns:   3                 │
                          │ cost:    $0.0847           │
                          └──────────────────────────┘
                                    ↑
═══════════════════════════════════════════════════════  ← PaneHeader
```

No arrow/caret. No drop-shadow (Variant B has no shadows — use border only). `border: 1px solid var(--border)`.

**Popover direction rationale:** Status bar pops upward (A10) because it is at the bottom of the viewport. PaneHeaders are mid-screen; popovers open downward to avoid covering the terminal content above the header.

## Component Inventory

| Component | File | Responsibility |
|-----------|------|----------------|
| `BudgetBar` | `src/grid/BudgetBar.tsx` | Inline header segment: cost text + bar track + bar fill. Props: `budget: BudgetState`. Click opens BudgetPopover. Conditionally rendered by PaneHeader (agent panes only). |
| `BudgetPopover` | `src/grid/BudgetPopover.tsx` | Popover card content: 5-row detail table (tokens, limit, model, turns, cost). Reuses A10 `<Popover>` for dismiss/positioning logic. Read-only. |
| `budgetSignal` | (local in `PaneComponent.tsx`) | Per-pane `createSignal<BudgetState | null>(null)`. Updated by `BudgetUpdate` PTY event handler. Passed as prop to PaneHeader → BudgetBar. No global store (D-12). |

### BudgetState Interface

```typescript
interface BudgetState {
  tokens_used:   number;           // cumulative tokens consumed
  token_limit:   number | null;    // null = unlimited (D-07: no bar)
  cost_usd:      number;           // cumulative cost in USD
  iteration:     number;           // agent turn count
  model:         string;           // e.g. "claude-sonnet-4-5"
}
```

### BudgetBar Props Interface

```typescript
interface BudgetBarProps {
  budget: BudgetState;
  onClickDetail: () => void;  // opens BudgetPopover
}
```

### BudgetPopover Props Interface

```typescript
interface BudgetPopoverProps {
  budget:  BudgetState;
  anchor:  HTMLElement;       // BudgetBar button element (for A10 Popover positioning)
  onClose: () => void;
}
```

## Budget Bar Fill Logic

### Threshold Computation

```
pct = token_limit != null ? (tokens_used / token_limit) * 100 : 0

fillColor =
  pct < 70  → var(--accent-green)
  pct < 90  → var(--accent-amber)
  pct >= 90 → var(--accent-red)

fillWidth = clamp(pct, 0, 100)%   // clamped, never overflows track
```

### CSS Transition (D-13)

```css
.budget-bar-fill {
  transition: width 150ms ease-out;
}

@media (prefers-reduced-motion: reduce) {
  .budget-bar-fill {
    transition: none;
  }
}
```

Cost text updates instantly (no transition). Only bar fill width transitions.

## Popover Content Specs

### Budget Detail Popover

Width: 220px. Height: content-driven (fixed 5 rows + header row). No scroll.

```
┌─────────────────────────────┐

├─────────────────────────────┤
│ tokens:  12,450 / 20,000    │  ← 24px rows; label --fg-3 11px, value --fg-1 12px
│ limit:   20,000 tokens      │

│ turns:   3                  │
│ cost:    $0.0847            │

```

| Element | Style |
|---------|-------|
| Header row | 11px `--font-mono` weight 500, `--fg-2`, `--bg-3`, 24px height, 8px horizontal padding |
| Field label | 11px `--font-mono` `--fg-3`, right-aligned, fixed 44px width |
| Field value | 12px `--font-mono` `--fg-1`, left-aligned, `text-overflow: ellipsis`, max-width calc |
| Row height | 24px each |
| Row padding | 0 8px (sm) |
| Row separator | 1px `--border` between rows |

┌─────────────────────────────┐

Expanded bar height: **8px** (2×4px grid). No transition on the popover bar — it is a snapshot at open time.

## Copywriting Contract

### Inline Header (BudgetBar)

| State | Display |
|-------|---------|
| Agent pane, first OSC not yet received | *(nothing rendered)* |
| Agent pane, unlimited budget | `$0.08` (cost only, no bar) |
| Agent pane, budget cap set | `$0.08 [progress bar]` |
| Cost < $0.01 | `$0.0847` — 4 decimal places |
| Cost $0.01–$0.99 | `$0.08` — 2 decimal places |
| Cost $1.00–$9.99 | `$1.23` — 2 decimal places |
| Cost ≥ $10.00 | `$12.34` — 2 decimal places |
| Cost ≥ $100.00 | `$123` — 0 decimal places (too wide for 44px max) |

**Cost format rule:** 4 decimal places for `cost_usd < 0.01`; 2 decimal places for `cost_usd < 100`; 0 decimal places for `cost_usd >= 100`. Always `$` prefix. No thousands separator within the header label (space constraints).

### Popover Field Labels and Values

| Field | Label | Value Format |
|-------|-------|-------------|
| Tokens used + limit | `tokens:` | `{used} / {limit}` with thousands separator (e.g. `12,450 / 20,000`) |
| Token limit (standalone) | `limit:` | `{limit} tokens` (hidden when null) |
| Model name | `model:` | Raw model string from payload (e.g. `claude-sonnet-4-5`). Truncate to 24ch with ellipsis. |
| Agent turn count | `turns:` | Integer (e.g. `3`) |
| Cost | `cost:` | Full precision: `$0.0847` (4dp when < $0.01), `$1.23` (2dp otherwise). No truncation in popover. |

### Popover Heading

| Element | Copy |
|---------|------|
| Popover heading | `Budget Detail` |
| Unlimited budget (no limit row) | `limit:` row omitted entirely — no placeholder text |

## Interaction Contracts

### BudgetBar Click (D-09, D-10)

| Trigger | Action |
|---------|--------|
| Click BudgetBar segment | Open BudgetPopover anchored to BudgetBar button; close any other open popover (A10 D-01 one-at-a-time rule) |
| Click outside popover | Dismiss popover |
| `Esc` key | Dismiss popover; return focus to pane terminal |
| Click BudgetBar when popover already open | Close popover (toggle behavior) |
| BudgetBar absent (shell pane or no data) | No click target — no interaction |

**Read-only popover:** No rows in the popover are actionable. No buttons, no links, no form fields. Dismiss only.

### Budget Signal Update (D-12, D-14)

| Event | Action |
|-------|--------|
| `BudgetUpdate` PTY event received | Update per-pane local `budget` signal with new `BudgetState`. No debounce (D-14). |
| Pane unmounts | Signal garbage-collected. No persistence (D-11). |
| App restart / pane remounted | Signal starts as `null`. Header shows nothing until first OSC. |
| `token_limit` absent in payload | Render cost only. No bar. |

### PaneHeader Rendering Gate

```
Show BudgetBar when:
  props.agentConfig != null   // F1 D-06: agentConfig prop signals agent pane
  && budget() != null         // first OSC received
```

The `agentConfig` prop on PaneComponent (F1 D-06) is the sole gate. Shell panes have no `agentConfig` — BudgetBar is never mounted for them.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard access | BudgetBar is a `<button>` element — reachable by Tab, activated by Enter/Space |
| Focus visible | `box-shadow: inset 0 0 0 1px var(--focus)` on keyboard focus |
| Screen reader — bar | `aria-label` on BudgetBar button includes current state, e.g. `aria-label="Budget: $0.08, 62% of 20,000 tokens. Open detail"` |
| Screen reader — no limit | `aria-label="Budget: $0.08. Open detail"` |
| Screen reader — popover | `role="dialog"` on BudgetPopover, `aria-label="Budget Detail"` |
| Popover rows | `role="list"` not needed (static table, not interactive list) |
| Reduced motion | `prefers-reduced-motion: reduce` removes bar width transition (D-13) |
| Bar color alone | Bar fill color changes by threshold — do not rely on color alone. `aria-label` on button reports percentage textually. |
