<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE byte stream → typed events | Untrusted/malformed server frames cross into Decode; must not crash or hang. |
| context cancel → server turn cancel | Client disconnect must cancel the in-flight turn (PROTOCOL §8) and not leak local resources. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.3-04-01 | DoS | Malformed SSE frame / unknown event `type` | mitigate | Per-frame `Decode()` returns a typed error for unknown types (Plan 02); parser tolerates ping/comment/id lines; a decode error is surfaced (not a panic). |
| T-V13.3-04-02 | DoS (resource exhaustion) | Goroutine/connection leak on cancel | mitigate | `http.NewRequestWithContext` tears down the TCP read on cancel; `defer resp.Body.Close()` + `defer close(ch)` guarantee teardown; verified by the goroutine-baseline assertion in `TestIntegrationSSECancel`. |
| T-V13.3-04-03 | DoS | Oversized/never-ending `data` line | mitigate | `bufio` reader bounded by the context deadline; cancel ends the read. (Note any buffer-size cap chosen for the scanner in the SUMMARY.) |
| T-V13.3-04-04 | Spoofing | SSE GET without bearer | mitigate | SSE GET routes through the Plan-03 bearer chokepoint; wrong token → 401 returned from `Events()` before any stream. |
| T-V13.3-04-05 | Information Disclosure | Token in stream-open error | mitigate | Open errors are `VossError` (status+detail only); token not formatted into the error. |
</threat_model>
