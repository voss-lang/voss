<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK → spawned child process | Interpreter path + args determine what executes; orphaned children hold ports. |
| stdin pipe heartbeat | The server self-terminates on stdin EOF; mismanaging the pipe can DoS or zombie the server. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.3-05-01 | Tampering/Elevation | Interpreter resolution / command injection | mitigate | Interpreter comes only from the fixed `VOSS_PYTHON → .venv/bin/python → python3` chain; exec args are fixed literals (`-m voss.cli serve --port 0`), no shell, no caller-supplied path interpolation → no command/path injection. |
| T-V13.3-05-02 | DoS | Orphan `voss serve` holding the port after Close/cancel | mitigate | `Close()` closes stdinW (EOF supervision) + `cmd.Process.Kill()` + `cmd.Wait()` reaps the zombie on every exit path; `exec.CommandContext` also kills on ctx cancel; PID-verified by `TestSpawnNoOrphan`. |
| T-V13.3-05-03 | DoS | Spawn hang on a bad interpreter / slow start | mitigate | `readHandshake` 20s timeout → typed `SpawnError`; no unbounded wait (`TestSpawnBadInterpreter`). |
| T-V13.3-05-04 | DoS | Server zombie if stdin write end closed early | mitigate | `stdinW` held in the client's `spawnState` and closed ONLY in `Close()` (Pitfall 5); never written. |
| T-V13.3-05-05 | Information Disclosure | Token from the handshake line leaking to stderr/logs | mitigate | Stdout (handshake) is parsed in-process, not logged; only stderr is inherited (server diagnostics, no token); token stays in the `Client` struct. |
</threat_model>
