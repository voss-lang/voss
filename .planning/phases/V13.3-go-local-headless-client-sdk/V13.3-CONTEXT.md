<decisions>
## Implementation Decisions

### Module location + import path
- **D-01:** Module lives at **`sdk/go/`** — grouped under `sdk/` alongside V13.1's `sdk/typescript/`, giving a consistent multi-language SDK home. (Diverges from V13.2's `crates/voss-sdk` placement, which followed Cargo-workspace convention; `sdk/<lang>` grouping wins for the non-Rust clients.)
- **D-02:** Go module path **`github.com/vosslang/voss/sdk/go`** (monorepo submodule with its own `go.mod`). Matches the `vosslang` org used by the npm packages (`@vosslang/...`). Publish / module-proxy discoverability stays **deferred** (SPEC out-of-scope) — the path is fixed now so imports and the eventual publish align.

### SSE-delivery idiom + dependency policy
- **D-03:** SSE consumer delivers events on a **receive channel (`<-chan Event`)**; the caller cancels via **`context.Context`**. Cancelling/closing disconnects, which cancels the in-flight turn (disconnect-cancels, PROTOCOL §8). Channel idiom composes with `select`/goroutines for concurrent headless workflows. (`iter.Seq2` range-over-func and callback handler both REJECTED — channel is the most composable for the CI-bot audience and avoids a hard Go 1.23 floor.)
- **D-04:** **stdlib `net/http` only** — no third-party SSE/HTTP library. SSE framing (`event:` / `data:` lines) is hand-parsed in the thin client, matching the carry-forward "codegen = types only, client wiring hand-written" pattern (V13.2 D-02). Bearer auth + loopback base URL attached on every request including the SSE GET.

### Codegen integration + drift gate
- **D-05:** Types generated with **`oapi-codegen`** reading V13.1's committed **`contracts/openapi.json`**. Because `app.py:_force_event_schema` forces `EventEnvelope` into OpenAPI components, the single OpenAPI pass captures BOTH the REST request/response shapes AND the event-union members.
- **D-06:** Generated Go types are **CHECKED IN** (committed), not build-time-only — keeps the module buildable/inspectable without re-running codegen and enables the byte-identical drift check (carry-forward from V13.1 D-11 / V13.2 D-06).
- **D-07:** Go drift gate = a **`go test`** that regenerates types from the committed `contracts/openapi.json` and fails on a non-empty `git diff` of the generated file (mirrors V13.2 D-08). Wired via a `go generate` directive + an `oapi-codegen` version pinned through `tools.go`. V13.1's **Python snapshot/drift gate stays upstream** (owns the any-diff re-export); the Go gate is **additive** and only asserts Go-side byte-identity.
- **D-08:** **Hard execution dependency on V13.1, NOT V13** — V13.1 OWNS `contracts/{openapi.json,events.schema.json}` + the deterministic exporter + the Python drift gate. V13 is docs-only (strategy/matrix/tiers) and is not a build dependency. The snapshot artifacts do not exist on disk yet (V13.1 unexecuted as of 2026-06-07); V13.3 may be planned ahead but execution blocks on V13.1 shipping `contracts/`.

### Claude's Discretion
- **Event-model representation** (deselected for discussion → research/planning discretion within the SPEC fidelity bar): how the 21 members map to Go given oapi-codegen's awkward union handling — likely an `EventEnvelope{ Type string; Data json.RawMessage }` + a typed-payload decode helper + a `type`-switch, but the exact shape is for research/planning. Hard constraints from SPEC req 2: all 21 members decodable into typed values; an unknown/future `type` yields a distinguishable error/sentinel, never a silent drop.
- **Codegen tool flavor** (oapi-codegen is locked as the tool, D-05) but the exact config (types-only vs client gen, `oapi-codegen` `-generate` flags) is discretion — note the REST client wiring is hand-written regardless (D-04).
- Public API ergonomics (constructor names for spawn vs attach, error type shape, package boundaries within `sdk/go/`), Go version floor (pragmatic, channel idiom does not force ≥1.23), and `VOSS_SERVE_FAKE_TURN` test-fixture wiring — left to research/planning within SPEC acceptance criteria.

</decisions>

## Existing Code Insights

### Reusable Assets
- `VOSS_SERVE_FAKE_TURN` env seam (server `_run_turn`) — emits a canned turn over the real event/SSE path with no provider creds. The integration-test driver for CI (same seam V13.1/V13.2 use).

- `crates/voss-tui` net/server/event modules — proven behavioral reference (Rust) for the Go REST client + spawn supervisor; not imported, mirrored.

### Established Patterns
- Handshake-then-bind (`serve.py`): spawn `voss serve --port 0` with stdin PIPED (EOF supervision), read one stdout JSON line `{v,port,token}`, then `http://127.0.0.1:<port>` + `Authorization: Bearer <token>` on every request incl. the SSE GET. (Spawn path; the attach path takes a caller-supplied URL+token instead.)
- SSE framing per PROTOCOL §6: `event: <type>` / `data: {...}`; first event `server.connected`, turn end `session.idle`; client disconnect cancels the in-flight turn (§8). Hand-parsed in Go (D-04).
- Non-2xx → typed error carrying the HTTP status + server `{v, detail}` body.
- Permission round-trip (PROTOCOL §7): `permission.updated {id,tool_name,args,dimension}` → reply `POST /session/:id/permission` choice ∈ `a|A|d` (+ scope).
- Checked-in generated types + per-language regenerate-and-git-diff drift gate (D-06/D-07; carry-forward).

### Integration Points
- REST endpoints (`app.py`): `POST/GET /session`, `GET /sessions/saved`, `GET/DELETE /session/:id`, `POST /session/:id/message`, `POST /session/:id/abort`, `GET /session/:id/cost`, `POST /session/:id/permission`, `GET /session/:id/events` (SSE), `GET /doctor`.
- New `sdk/go/go.mod` (module `github.com/vosslang/voss/sdk/go`) — a fresh module; the repo currently has no `go.mod`.

<specifics>
## Specific Ideas

- Spawn constructor honors interpreter resolution `VOSS_PYTHON` → repo `.venv/bin/python` → `python3` (matches the Rust supervisor). `Close()`/context cancel must leave no orphan `voss serve` (PID-verified per SPEC AC).
- Attach constructor owns no child process — caller supplies base URL + token (or a captured handshake line); used by CI bots connecting to a long-lived serve.
- Auth concern split is deliberate: SDK owns the **local-server handshake bearer token** only; not provider creds, not the JSON-RPC bridge transport.
- No-FFI guard is testable: `grep` of `sdk/go/` shows no `cgo`/FFI import and no reimplemented orchestration types (SPEC req 7 / AC).

</specifics>
