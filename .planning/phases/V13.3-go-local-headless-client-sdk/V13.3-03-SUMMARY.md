## spawnState contract (Plan 03 → Plan 05)

`spawnState` is defined as a STUB in `client.go` with exactly:

```go
type spawnState struct {
	cmd    *exec.Cmd
	stdinW io.WriteCloser
	pid    int
}
```

The attach path leaves `Client.spawn` nil. **Plan 05's `spawn.go` CONSTRUCTS and FILLS this struct and owns the teardown (kill + reap + stdin close) in `Close()`; it must NOT re-declare the type.** `client.go` compiles standalone today (no `undefined: spawnState`).

## Status-code map (RESEARCH Pitfall 1)

`do(req, want, out)` asserts the exact success code: CreateSession→`StatusCreated` (201), PostMessage/Abort→`StatusAccepted` (202), DeleteSession→`StatusNoContent` (204), GETs→`StatusOK` (200). All four constants referenced in rest.go.
