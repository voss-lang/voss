## What shipped

- `sdk/go/errors_test.go` (new) — `TestVossError`.
- `sdk/go/events_test.go` — green: `TestDecodeAllMembers` (21-entry table, field spot-checks on final/session.idle/permission.updated/principles_overflow/budget.updated/gate.updated) + `TestDecodeUnknownType` (`errors.As`).

## Key design decision

The 21 event structs already exist in `types.gen.go` (same `package voss`), so they **cannot** be redefined. `events.go` instead **adds `eventType()` methods to the generated structs** (value receivers) so the values returned by the generated `AsX()` accessors satisfy `TypedEvent` directly. `Decode` uses the two-line `v, err := env.Event.AsX(); return v, err` form (Go won't implicitly convert a concrete type to the interface in a multi-value passthrough `return`). Discriminator shape per Plan 01: `EventEnvelope_Event.Discriminator() (string, error)` — used directly, no fallback.
