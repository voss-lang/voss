# Phase M9 — UI Design Contract (Terminal UI)

> Visual + interaction contract for the Voss TUI shell. Generated by gsd-ui-researcher. All web-UI dimensions (spacing, typography, color, registry) are translated to terminal-UI equivalents. The TUI is the visible product surface for Voss's language primitives — probable values, ctx budgets, spawn/gather. See `.planning/notes/voss-agent-unfair-advantage.md` for thesis.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (no web design system) | n/a — terminal UI |
| Preset | not applicable | n/a |
| Library | **Textual** (strong default; planner may swap to prompt_toolkit only if Textual fails the Windows-console gate from CONTEXT.md) | CONTEXT Claude's Discretion |
| Renderer fallback | `rich`-based `render.py` (existing `PlainRenderer` / `JsonRenderer`) | `voss/harness/render.py` |
| Glyph vocabulary (locked) | `▌` prompt, `❯` user-input, `⏵` tool-call, `⚠` warning, `·` separator. **No emoji.** Inherited from existing renderer to keep parity with `--plain`. | `voss/harness/render.py` line ~58 |
| Confidence bar glyph | `█` filled, `░` empty (e.g. `████████░░ 0.82`) — true single-cell width on all monospace fonts | UI default |
| Budget meter glyph | `▰` filled, `▱` empty (e.g. `▰▰▰▰▰▱▱▱▱▱  2.1k / 4.0k`) — distinguishable from confidence at a glance | UI default |
| Sub-agent marker | `└─` for nested spawn, `├─` for sibling gather entry | UI default |
| Diff marker glyphs | `+` add line, `-` remove line, `~` modified-line marker for per-hunk header | UI default |
| Branch / fork marker | `⎇` (U+2387) in session list when a turn has children; falls back to `+` if `--no-unicode` set | UI default |
| Color depth target | 256-color minimum; true-color (24-bit) when `COLORTERM=truecolor` | Textual default |
| Color depth fallback | 16-color terminals: dim/bold-only styling; monochrome (`NO_COLOR=1` honored): all signal carried by glyphs + bold/dim/italic, never color alone | Accessibility default |
| Unicode fallback | `--no-unicode` flag downgrades box-drawing + confidence/budget glyphs to ASCII (`#` / `.` / `|` / `+` / `-`) | UI default |
| Theme detection | Honor `$VOSS_THEME` (`dark` / `light` / `auto`); `auto` reads `$COLORFGBG` if present, else assumes dark | CONTEXT marks theming deferred — minimal viable only |
| Min terminal size | **80 cols × 24 rows**. Below this, TUI exits with stderr message + suggests `--plain`. | UI requirement (diff-renderer constraint in CONTEXT) |
| Target terminal size | 120 cols × 40 rows (drives layout proportions below) | UI default |

## Layout / Spacing (Character Cells)

Spacing is measured in **character cells** (rows × columns), not pixels. All paddings and margins are integers; no fractional cells.

### Region grid (locked from CONTEXT.md)

```
┌──────────────────────────────────────────────────────────────────┐
│ HEADER  (1 row, full width)                                      │   row 0
├──────────────────────────────────────────────┬───────────────────┤
│                                              │                   │
│  MAIN PANE — turn history (scrollable)       │  SIDE PANEL       │   rows 1..N-3
│                                              │  spawn/gather     │   (variable)
│                                              │  sub-agents       │
│                                              │  (hidden when     │
│                                              │   none active)    │
│                                              │                   │
├──────────────────────────────────────────────┴───────────────────┤
│ STATUS LINE  (1 row, full width)                                 │   row N-2
├──────────────────────────────────────────────────────────────────┤
│ INPUT BAR  (1–3 rows, grows with multi-line input)               │   row N-1..
└──────────────────────────────────────────────────────────────────┘

Modal pane (diff approval, permission prompt) overlays MAIN + SIDE,
centered, max width 100 cols, min margin 4 cols left/right.
```

### Region dimensions

| Region | Height (rows) | Width (cols) | Padding | Notes |
|--------|---------------|--------------|---------|-------|
| Header | 1 fixed | full | `1` left/right | Single-line; never wraps. Truncate with `…` if overflow. |
| Main pane | flex (fills available) | `60%` of cols, **min 60** | `1` top/bottom, `2` left/right | Scrollable. Auto-scrolls to tail on new turn unless user scrolled up (sticky-to-top mode). |
| Side panel | flex (matches main) | `40%` of cols, **min 28, max 50** | `1` top/bottom, `2` left/right | Hidden (collapsed to 0 cols) when no active `spawn`. Border on left edge: `│`. |
| Status line | 1 fixed | full | `1` left/right | Reverse-video background; tokens, cost, ctx-pct, model. |
| Input bar | 1 base, grows up to **5 rows** for multi-line | full | `1` left/right | Prompt glyph `▌` at col 0. Shift+Enter inserts newline; Enter submits. |
| Modal overlay | flex (max 80% of rows) | flex (max 100 cols, **min 60**) | `1` row top/bottom, `2` col left/right | Drop-shadow not used (terminal limitation). Border `─│┌┐└┘`. Dims/desaturates layers below. |
| Slash palette popup | up to **8 rows** | matches input bar | `1` left/right | Anchored above input bar, shows ranked commands as user types `/`. |

### Spacing scale (character cells)

Multiples of 1 cell only. Web "8-point grid" does not apply.

| Token | Value | Usage |
|-------|-------|-------|
| `tight` | 0 cells | Adjacent labels within a single status field |
| `xs` | 1 cell | Between glyph and following text (`⏵ fs_read`) |
| `sm` | 2 cells | Standard inline padding inside a region |
| `md` | 1 row / 2 cells | Between turns in main pane; between fields in modal |
| `lg` | 2 rows | Between top-level sections inside a modal (header → body → footer) |

Exceptions: none.

## Typography (Terminal Text Roles)

Terminal "typography" = combinations of **bold / italic / dim / underline / reverse-video**, plus **truncation / wrap rules**. No web fonts, no `px`, no line-height ratios — only character cells.

| Role | Style | Truncation rule | Used for |
|------|-------|-----------------|----------|
| **Body** | regular (no SGR weight) | Wrap at region width − padding, word-boundary | Turn content, agent reasoning, tool output |
| **Emphasis** | bold | Same wrap as body | Section headers inside a turn, focused list item, user message prefix |
| **Dim** | dim (SGR 2) | Same wrap as body | Metadata: timestamps, costs, glyph descriptors (`· 0.12s · $0.004`) |
| **Code / paths** | regular, monospace assumed | Truncate middle with `…`; never wrap mid-token | File paths, `.voss` snippets, tool arg values |
| **Signal-good** | regular + green | n/a (status only) | Tool success state, confidence ≥ 0.85 |
| **Signal-warn** | bold + yellow | n/a | Confidence 0.50–0.84, soft validation failure, budget ≥ 75% drained |
| **Signal-error** | bold + red | n/a | Tool error, permission denial, budget exhausted, diff rejected |
| **Reserved (italic)** | italic | n/a | Reserved for future use (e.g. agent-internal commentary). Do NOT use in M9 — many terminals render italic as reverse, causing collision with focused-row style. |

### Three style tiers (locked)

The contract declares **exactly three weight/style tiers**: regular, bold, dim. Color is a separate axis. Italic and underline are reserved (not used in M9).

### Max line widths

| Context | Max width | Rationale |
|---------|-----------|-----------|
| Body text in main pane | region width − 4 (padding) | Reader comfort; no hard cap above terminal width |
| Code blocks / diff lines | region width − 4 | Horizontal scroll preferred over wrap for code |
| Status line fields | each field ≤ 24 cells before `·` separator | Prevents jitter on cost/token updates |
| Modal title | 60 cells | Single-line headline |

### Confidence / budget rendering rule (locked)

- Confidence bar is **always 10 cells wide** (e.g. `████████░░`) plus a 4-cell numeric (`0.82`). Total 14 cells. Always followed by 1-cell padding.
- Budget meter is **always 10 cells wide** plus numeric `2.1k / 4.0k`. Total 18 cells. Always followed by 1-cell padding.
- These widths are fixed so values are scannable across stacked turns without alignment jitter.

## Color Contract (256-color / Truecolor + 16-color Fallback)

Voss TUI uses a **deliberately small palette** so signals stay legible. 60/30/10 split is interpreted as: 60% default terminal foreground/background (no color applied), 30% structural dim/bold, 10% signal accent.

| Role | Dark theme (truecolor) | Dark (16-color) | Light theme (truecolor) | Light (16-color) | Usage |
|------|------------------------|-----------------|-------------------------|------------------|-------|
| **Dominant (60%)** | terminal default fg/bg | default | terminal default fg/bg | default | Body text. We do NOT force a background — respect the user's terminal. |
| **Secondary (30%)** | `#888888` (dim grey) | SGR 2 (dim) | `#666666` | SGR 2 (dim) | Borders, separators, timestamps, glyph hints, status line bg accent |
| **Accent (10%)** | `#5FAFFF` (cyan-blue) | `cyan` (SGR 36) | `#005F87` | `blue` (SGR 34) | **Reserved-for list below** |
| **Signal-good** | `#5FD75F` (green) | `green` (SGR 32) | `#008700` | `green` (SGR 32) | Tool success, confidence ≥ 0.85, accepted diff hunk |
| **Signal-warn** | `#FFD75F` (amber) | `yellow` (SGR 33) | `#AF8700` | `yellow` (SGR 33) | Confidence 0.50–0.84, budget 75–99% drained, permission prompt |
| **Signal-error** | `#FF5F5F` (red) | `red` (SGR 31) | `#AF0000` | `red` (SGR 31) | Tool failure, budget exhausted, rejected diff, fatal validation |
| **Focus highlight** | reverse-video on accent | reverse | reverse-video on accent | reverse | Selected slash-palette row, focused modal button, focused diff hunk |

### Accent reserved for (explicit allow-list)

The accent color (`#5FAFFF` / cyan-blue) is used **only** for these elements. Anything else uses default fg or dim.

1. The user-input glyph `▌` at the input-bar prompt.
2. The session id in the header.
3. The current model name in the status line.
4. The active sub-agent name banner in the side panel header.
5. The current selection in the slash-command palette (combined with reverse-video).
6. Confidence bars at **the agent's final confidence value** (per-step bars stay grey-on-default).

NOT permitted to use accent: arbitrary text, tool names, file paths, timestamps, costs, ordinary borders, or "all interactive elements."

### Monochrome / `NO_COLOR=1`

When `NO_COLOR` is set or terminal reports 2-color capability:
- All color removed.
- Confidence/budget tiers carried by **glyph density alone** (filled vs empty cells).
- Signal states carried by **bold (error/warn) vs dim (good is regular weight) + leading glyph** (`⚠` warn, `✗`→`x` error in `--no-unicode`).
- Reverse-video still used for focus.

## Copywriting Contract

All copy uses **plain American English, sentence case, no exclamation marks**. Imperative voice for prompts. No marketing tone. Path-references are wrapped in backticks in body text and rendered as `code` style.

### Locked strings

| Element | Copy |
|---------|------|
| **App banner (TTY start)** | `voss · agent\n{model} · {cwd} · {git_status}\n\nType a task, or /help.` (inherited verbatim from existing `TtyRenderer.banner`) |
| **Input prompt** | `▌ ` (glyph + single space) — no text label |
| **Primary CTA (slash command discovery)** | `Type / for commands · ? for help` (shown dim in status line when input bar is empty) |
| **Empty main pane (fresh session, no turns yet)** | Heading: `No turns yet`<br>Body: `Type a task below to start. Use / for commands, ? for help.` |
| **Empty session list (`voss sessions` / `/sessions`)** | Heading: `No sessions in this repo`<br>Body: `Run voss do "<task>" to create one.` |
| **Empty side panel (no sub-agents)** | Side panel hidden entirely (collapsed to 0 cols). No placeholder copy. |
| **Loading / thinking** | `⏵ thinking…` (dim, animated with `…` → `.` → `..` → `…` cycle, 400ms per frame) |
| **Tool call in progress** | `⏵ {tool_name}  {short_args}` — args truncated to 60 cells with `…` |
| **Tool call success** | Same line, suffix ` · ok` (signal-good) |
| **Tool call error** | Same line, suffix ` · failed: {message}` (signal-error). Message truncated to 80 cells; full error one-tap viewable via `Enter` on the line. |
| **Generic error toast** | `⚠ {summary}` followed by single-line hint. Example: `⚠ provider request failed · check voss doctor` |
| **Budget warning (≥ 75% drained)** | Status line budget field switches to signal-warn color. No modal interruption. |
| **Budget exhausted** | Modal: heading `Budget exhausted`<br>Body: `Turn stopped at {tokens} / {limit} tokens. Continue with a new budget, or end the turn.`<br>Buttons: `[c] Continue +2000` · `[e] End turn` · `[Esc] Cancel` |
| **Permission prompt (deny / allow once / allow always)** | Modal heading: `Permission required`<br>Body: `Tool {tool_name} wants to {action_verb} {target}.`<br>Buttons: `[a] Allow once · [A] Allow always · [d] Deny · [Esc] Deny` |
| **Diff approval modal heading** | `Review changes · {n} hunks · {file_count} files` |
| **Diff hunk action labels** | `[y] Accept · [n] Reject · [s] Skip · [a] Accept all · [q] Reject all · [Esc] Cancel review` |
| **Diff hunk accepted toast** | `✓ hunk accepted` (signal-good, dim) — appears in status line for 1500ms |
| **Diff hunk rejected toast** | `✗ hunk rejected` (signal-warn, dim) — appears in status line for 1500ms |
| **Diff review cancelled** | Modal closes. Status line: `Review cancelled · no changes applied` for 2000ms. |
| **Fork-from-turn confirmation** | Modal heading: `Fork session from turn {n}?`<br>Body: `Creates a new session starting from this turn. The current session keeps its history.`<br>Buttons: `[Enter] Fork · [Esc] Cancel` |
| **Session resume opens** | Status line flashes: `Resumed {session_id} · {n} turns` for 2000ms. |
| **`--plain` auto-fallback notice** | Stderr only: `voss: non-TTY stdout detected · using --plain mode` |
| **Min-size guard (terminal too small)** | Stderr + exit code 2: `voss: terminal must be at least 80×24 (current: {cols}×{rows}). Resize or use --plain.` |
| **Windows console limitation hit** | Stderr + exit 0 with `--plain` fallback (planner picks: hard-block vs soft-degrade per CONTEXT). Notice copy: `voss: Windows console missing capability {cap} · using --plain mode` |
| **Help overlay header** | `voss tui · keys + commands` |

### Destructive actions inventory

Three destructive actions exist in this phase. All require confirmation; none auto-fire.

| Action | Where | Confirmation pattern |
|--------|-------|----------------------|
| Apply file edit (per hunk) | Diff approval modal | Per-hunk explicit `y` keypress. No bulk accept-all without distinct `[a]` press. Pressing `Esc` cancels the entire review with zero hunks applied. |
| Reject entire diff (loses agent's proposed edit) | Diff approval modal `[q]` | Single keypress; agent receives "all hunks rejected" signal and may re-plan. Not asymmetric with accept — symmetry signals diff approval is the user's primary lever. |
| Run shell command via `shell_run` | Permission prompt modal | Three-choice: allow-once / allow-always / deny. Default action on `Esc` is **deny**. |

Notes:
- Session fork is **not** destructive — original session is preserved. Confirmation is informational, not protective.
- Slash commands themselves are not destructive at the UI level; their mutating bit (`SlashCommand.mutating=True`) routes through the same permission prompt as tool calls.

### Tone and writing rules

- **Sentence case** for all UI strings. Never Title Case.
- **No exclamation marks.** Voss is a tool, not a cheerleader.
- **Imperative** for actions ("Type a task", "Run voss doctor"), not "You can…" hedging.
- **Backtick file paths and command names** in body copy, render as `code` style.
- **Numeric units**: tokens as `2.1k` / `12.4k` (one decimal, k-suffix at 1000+); cost as `$0.004` (3 decimals minimum); durations as `0.12s` / `2.4s` / `1m 12s`.
- **No "Voss thinks…" or "Voss says…" personification.** Use `⏵ thinking…` and direct content.

## Interaction Contract

### Focus model

- **Single focused region** at any time. Tab/Shift-Tab cycle: Input Bar → Main Pane → Side Panel (skipped if hidden) → Modal (if open, captures focus exclusively).
- Focused region indicated by accent-colored border on left edge (col 0 of region uses `┃` thick bar instead of `│`).
- Input bar is **default focus** on app start and after each turn completes.

### Keybindings (locked structure; full table refinable by planner)

| Key | Context | Action |
|-----|---------|--------|
| `Tab` / `Shift+Tab` | global (no modal) | Cycle focus regions |
| `Enter` | input bar | Submit task |
| `Shift+Enter` | input bar | Insert newline |
| `/` | input bar (when empty) | Open slash palette |
| `Esc` | any modal | Close modal (deny / cancel — see destructive table) |
| `?` | global (no modal) | Open help overlay |
| `j` / `k` | main pane focused | Scroll one row down / up |
| `Ctrl+d` / `Ctrl+u` | main pane focused | Scroll half-page down / up |
| `g` / `G` | main pane focused | Jump to top / bottom |
| `f` | main pane focused, turn highlighted | Fork session from this turn |
| `Ctrl+f` | main pane focused | Open in-pane search prompt (uses `/` glyph at status line, distinct from palette `/`) |
| `y` / `n` / `s` / `a` / `q` | diff modal | Accept / reject / skip / accept-all / reject-all hunk |
| `a` / `A` / `d` | permission modal | Allow-once / allow-always / deny |
| `Ctrl+c` | global | Interrupt current turn (sends SIGINT to provider call); second press exits app |
| `Ctrl+l` | global | Redraw screen (recovery from corrupted terminal state) |

Disambiguation rule (called out in CONTEXT): in the **input bar**, `/` opens the palette. In the **main pane**, `/` opens in-pane search via the status line (no collision because focus is mutually exclusive).

### Animation / motion budget

- No decorative animation. Only:
  - **Thinking indicator** — 3-dot cycle, 400ms/frame.
  - **Status-line toast** — fade timing is binary (visible for N ms, then removed). No alpha cross-fade.
  - **Confidence/budget bars** — instant update on event; no tween.
- Refresh rate cap: **20 fps** (50ms tick). Higher rates waste cycles in vendored Python on Windows console.

### Live workflow visualization (locked from CONTEXT)

The TUI is the **product face** of Voss primitives. Render contract:

| Primitive | Visual treatment | Source |
|-----------|------------------|--------|
| `probable<T>` | Inline confidence bar (10 cells + numeric) immediately after the value's label. Color tier by threshold: ≥0.85 good, 0.50–0.84 warn, <0.50 error. | `voss_runtime/probable.py` via recorder |
| `ctx(budget: N tokens)` | Budget meter in status line (10-cell `▰` bar + `{used}k / {total}k`). Drains live as tokens stream. Color tiers at 75% and 100%. | `voss_runtime/budget.py` via recorder |
| `within budget { } fallback { }` | When fallback branch fires, main pane shows a single line: `⤳ fallback triggered · {reason}` in signal-warn. | recorder event |
| `spawn` | Side panel opens (or new panel added) with sub-agent name accent-colored, own mini status line (tokens / cost), own scrollable body. | `voss_runtime/agent.py` via recorder |
| `gather` | Side panels collapse with a brief `✓ gathered · {n} results` line in main pane. | recorder event |
| `memory.episodic / .semantic / .working` | Not rendered in M9 main flow (memory browse panel is reserved for M8). When a memory write occurs, status line shows dim `· wrote {kind} memory` toast for 1500ms. | recorder event |

If recorder lacks an event for any primitive above, **the TUI degrades gracefully** (omits that visualization) rather than extending the runtime surface. CONTEXT locks this — runtime hook extension is out of scope.

## Headless / `--plain` Contract (Locked)

The TUI must not regress headless output. Both paths share the same event stream; only the renderer differs.

| Property | Contract |
|----------|----------|
| Exit codes | Byte-identical to current v0.1 `--plain` behavior. |
| Stdout shape | Byte-identical when `--plain` is set OR when stdout is non-TTY. |
| Stderr shape | TUI may write extra diagnostics to stderr (e.g. non-TTY fallback notice), but never to stdout. |
| Pipe semantics | `voss do "task" | grep …` works identically pre- and post-M9. |
| Auto-detect rule | If `not sys.stdout.isatty()` OR `--plain` flag OR `$VOSS_PLAIN=1`, render via existing `PlainRenderer` from `voss/harness/render.py`. No TUI bootstrapping cost paid. |
| Verification | `tests/harness/test_plain_parity.py` (planner-named) diffs stdout bytes of representative tasks pre- and post-M9. Required acceptance gate. |

## Acceptance Visual Checks (for ui-checker + ui-auditor)

When ui-checker validates this contract and ui-auditor compares implementation, these are the must-pass observations.

1. **80×24 minimum honored** — at 79×24 or 80×23, app exits with the locked stderr message; at 80×24 exact, all locked regions render without truncation of header glyphs or status fields.
2. **Glyph vocabulary held** — no emoji introduced; only the locked set (`▌ ❯ ⏵ ⚠ █ ░ ▰ ▱ └─ ├─ ⎇`) plus standard box-drawing.
3. **Accent allow-list held** — accent color appears only on the six elements in the allow-list. Auditor greps the styling for misuse.
4. **`--plain` parity** — stdout byte-diff against pre-M9 baseline is zero on the parity test set.
5. **NO_COLOR=1 honored** — TUI renders without ANSI color codes; confidence/budget tiers still distinguishable via glyph density + bold/dim.
6. **No new runtime hook added** — diff of `voss/harness/recorder.py` / `voss_runtime/*` shows additions only to consumers, not to the runtime event surface. (CONTEXT-locked constraint.)
7. **Destructive-action confirmations present** — diff approval requires per-hunk keypress; permission prompt has three explicit choices; budget-exhausted has a continuation modal.
8. **Empty states render** — fresh session, empty session list, and no-sub-agents states show the locked copy (or correctly collapse, in the sub-agent case).
9. **Help overlay reachable** — `?` from any non-modal state opens help.
10. **Reserved slash names not occupied** — palette does not register `/recall`, `/forget`, `/memory`, `/save` (reserved for M8).
