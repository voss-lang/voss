## Spacing Scale

This pane uses a **density-first** model. The 8-point grid applies to the chrome layer (header, banners). The terminal body has its own internal spacing governed by xterm.js font metrics.

| Token | Value | Usage in A2 |
|-------|-------|-------------|
| xs | 4px | Icon-to-text gap in header; separator pad |
| sm | 8px | Header horizontal padding (left/right); banner button vertical pad |
| md | 16px | Banner horizontal padding; exit-banner internal margin |
| lg | 24px | Not used in A2 — single pane has no section breaks |
| xl | 32px | Not used in A2 |
| 2xl | 48px | Not used in A2 |
| 3xl | 64px | Not used in A2 |

**Exceptions:**
- Pane header height: exactly **22px** (Variant B lock — does not conform to 4px multiples). Source: sketch 001 README + MANIFEST.md.
- Touch targets: banner buttons (`Restart`, `Send`, `Discard`) minimum height 28px to allow click accuracy, even though the row is visually compact.
- Header font: 12px (see Typography) — dense by design, not an error.

### 2. Pane Header (22px)

**Height:** Exactly 22px. Single horizontal line. No wrapping.
**Background:** `--bg-3` (`#1f232e`)
**Left/right padding:** 8px (`sm`)
**Font:** `--font-ui` (Inter), 12px, weight 500 for labels, weight 400 for separators/dim elements

**Element order (left to right, all on one line):**

```
[8px] ● [4px] 1 [4px] · [4px] voss [4px] · [4px] zsh [4px] · [4px] vim [4px] ···spacing··· → → → ⋯ [8px]
```

| Slot | Content | Color | Font | Notes |
|------|---------|-------|------|-------|
| Status dot `●` | Unicode `●` (U+25CF) | `--accent-green` running; `--accent-red` exited; `--fg-3` loading | 12px/500 | Indicates shell lifecycle state |
| Separator `·` | Unicode `·` (U+00B7) | `--fg-3` (`#444a5a`) | 12px/400 | Visual divider between slots |
| Pane index | `1` (single digit or two-digit) | `--fg-2` (`#6a7080`) | 12px/400 | A2 = always `1`; A3 assigns real indices |
| Separator `·` | | `--fg-3` | | |
| cwd basename | Last path component of working directory (e.g. `voss`) | `--fg-1` (`#aab0c0`) | 12px/500 | Truncate to max 24 chars with `…` on right if longer |
| Separator `·` | | `--fg-3` | | |
| Shell name | Binary name (e.g. `zsh`, `bash`, `fish`) | `--fg-2` | 12px/400 | From `$SHELL` basename |
| Separator `·` | | `--fg-3` | | |
| Process indicator | Foreground command (e.g. `vim`, `git`, or empty) | `--fg-0` when active command; `--fg-3` when at shell prompt | 12px/500 | See Process Indicator spec below |
| Flex spacer | Fills remaining width | — | — | `flex: 1` |
| Menu button `⋯` | Unicode `⋯` (U+22EF) | `--fg-2`; hover `--fg-0` | 12px/400 | Click target 28×22px minimum; shows context menu (A7 wires palette; A2 stub is acceptable) |

**Truncation rule for cwd:** When basename > 24 chars, truncate at 24 with a trailing `…`. Never wrap. The header is always exactly 22px tall.

**Process indicator slot population (D-07):**
- **Primary:** `Terminal.onTitleChange(title)` — xterm fires when shell emits OSC 0/2. Set process slot to `title`.
- **Fallback:** When no OSC title received since pane open (or > 2 seconds since last OSC update), Rust polls `tcgetpgrp(master_fd)` + `libproc` (macOS) / procfs (Linux) at 500ms intervals. Sets process slot to resolved command name.
- **At shell prompt:** process slot shows empty string or shell name (same as Shell name slot) — use `--fg-3` color to dim it.
- **Loading state (pre-first-output):** slot shows nothing; status dot `●` in `--fg-3`.

**Header click behavior:**
- Clicking anywhere in header (except `⋯`) focuses the pane — same as body click.
- `⋯` button: shows stub menu in A2 (close, fork, rename options may be no-ops). Full wiring in A7.

### 3. Terminal Body

**Component:** xterm.js `Terminal` instance (v5.5.0, pinned exact), Canvas renderer (`CanvasAddon` v0.7.0).
**Mount:** fills 100% of pane container height minus 22px header.

**xterm.js Terminal initialization contract:**

```typescript
new Terminal({
  scrollback: 10_000,
  fontFamily: '"JetBrains Mono", "SF Mono", "Menlo", ui-monospace, monospace',
  fontSize: 13,
  lineHeight: 1.5,
  theme: {
    background:       '#0a0b0e',  // --bg-0
    foreground:       '#e8eaf0',  // --fg-0
    cursor:           '#5a7cff',  // --focus
    cursorAccent:     '#0a0b0e',  // --bg-0 (cursor text inverse)
    selectionBackground: 'rgba(122, 162, 255, 0.30)',  // --accent-blue at 30% opacity
    black:            '#0a0b0e',
    brightBlack:      '#444a5a',  // --fg-3
    red:              '#e87b7b',  // --accent-red
    brightRed:        '#e87b7b',
    green:            '#6fd28f',  // --accent-green
    brightGreen:      '#6fd28f',
    yellow:           '#e8b86c',  // --accent-amber
    brightYellow:     '#e8b86c',
    blue:             '#7aa2ff',  // --accent-blue
    brightBlue:       '#7aa2ff',
    magenta:          '#c084d4',  // --accent-magenta
    brightMagenta:    '#c084d4',
    cyan:             '#6cc7d4',  // --accent-cyan
    brightCyan:       '#6cc7d4',
    white:            '#aab0c0',  // --fg-1
    brightWhite:      '#e8eaf0',  // --fg-0
  },
  cursorBlink: true,
  macOptionIsMeta: true,
  rightClickSelectsWord: false,
  allowProposedApi: false,
})
```

**Padding:** Internal terminal padding = 0px. xterm body fills container with no extra inset. The border is on the container, not inset.

**Scrollback:** 10,000 lines (configurable via settings in A8; A2 hardcodes 10k).

**Scrollbar:** slim, dark, consistent with `themes/default.css` scrollbar styles:
```css
::-webkit-scrollbar { width: 8px; }
::-webkit-scrollbar-track { background: transparent; }
::-webkit-scrollbar-thumb { background: var(--bg-3); border-radius: 4px; }
::-webkit-scrollbar-thumb:hover { background: var(--border-bright); }
```

**Selection highlight:** `rgba(122, 162, 255, 0.30)` — `--accent-blue` at 30% opacity. Applied as `theme.selectionBackground` in Terminal options above.

**Copy affordance (D-06):**
- Text selected → `⌘C` copies to clipboard; xterm's selection is cleared after copy.
- No selection → `⌘C` writes `\x03` (ETX) to PTY (SIGINT to foreground pgid via kernel TTY driver).
- This behavior is configurable (settings in A8); A2 ships the smart default.
- No visual copy-feedback toast or animation — density-first means no confirmation UI for copy.

**Link handling (PTY-05):**
- OSC 8 explicit links: `term.options.linkHandler.activate` calls Tauri `open_url` on `⌘+click`.
- URL auto-detect: `@xterm/addon-web-links` v0.12.0. Hover = `--accent-cyan` underline.
- File-path detection: custom `registerLinkProvider` with regex `/(\/[^\s'"]+|~\/[^\s'"]+|\.[./][^\s'"]+)/g`. `⌘+click` calls Tauri `open_path`.
- Link hover cursor: default browser pointer cursor (no custom cursor tokens needed).

**Alt-screen apps (PTY-08):** xterm.js handles alt-screen natively with `TERM=xterm-256color`. No special visual treatment in A2. Known cosmetic issue: alt-screen exit may leave duplicate lines in normal scrollback (xterm.js issue #802 — accepted defect in A2, not a blocker).

### 4. Shell-Exit Banner (PTY-07)

**Trigger:** Rust reader thread detects PTY EOF (child.try_wait() returns Some(status)). Emits exit event to frontend with exit code N.

**Placement:** Inline, positioned at the **bottom** of the terminal body area. It appears as a 28px bar that overlays the last line of terminal content (does not push content up). `position: absolute; bottom: 0; left: 0; right: 0;` within the pane container.

**Height:** 28px fixed.
**Background:** `--bg-3` (`#1f232e`) — same as header, visually brackets the pane.
**Border-top:** 1px solid `--border` (`#262b38`)

**Layout (left to right, single line, 8px horizontal padding):**

```
[8px] ● [4px] [exited 0]  ···spacer··· Restart [8px]
```

| Slot | Content | Color | Notes |
|------|---------|-------|-------|
| Status dot `●` | `●` Unicode | `--accent-green` if exit code 0; `--accent-amber` if code 1–127; `--accent-red` if code > 127 or signal | |
| Exit message | `[exited 0]` — exact copy, brackets included | `--fg-1` for code 0; `--accent-amber` for non-zero | Font: `--font-mono` 12px/400 |
| Spacer | flex: 1 | | |
| Restart button | `Restart` | See button spec below | |

**Exit copy rules:**
- Exit 0: `[exited 0]` — `--fg-1` color (neutral)
- Exit 1–127: `[exited N]` — `--accent-amber` color
- Exit > 127 or signal-killed: `[exited N]` — `--accent-red` color
- No other copy variants. Copy is exactly `[exited N]` with the actual integer.

**Restart button:**

| State | Background | Text color | Border |
|-------|-----------|-----------|--------|
| Default | `--bg-2` | `--accent-blue` (`#7aa2ff`) | 1px `--border` |
| Hover | `--bg-3` | `--fg-0` | 1px `--border-bright` |
| Focus (keyboard) | `--bg-3` | `--fg-0` | 1px `--focus` (`#5a7cff`) + inset focus ring |
| Active/press | `--bg-1` | `--accent-blue` | 1px `--border` |

Button dimensions: min-width 64px, height 22px, horizontal padding 12px, font `--font-ui` 12px/500. Border-radius: **0** (Variant B — no rounding).

**Restart behavior:** clicking Restart kills the dead PTY session, spawns a fresh PTY with the same `$SHELL` and `cwd`, clears the exit banner, and resumes normal terminal operation. Scrollback from the previous session is preserved in the buffer.

**Pane stays open:** The pane does NOT auto-close on shell exit. Ever. The banner persists until Restart is clicked or the user closes the pane manually (A3 delivers close; in A2 the window remains single-pane).

### 5. Paste-Guard Banner (PTY-04 / D-04 / D-05)

**Trigger:** DOM `paste` event fires on the xterm container with clipboard content containing ≥1 newline character (`\n`). Single-line pastes pass through directly without banner.

**Bypass:** `⌘⇧V` — intercepted in `customKeyEventHandler` before paste fires; sets a one-shot `bypassFlag = true` and lets paste proceed without the banner. No banner shown, no confirmation.

**Placement:** Inline, floating above terminal content. `position: absolute; bottom: 28px; left: 0; right: 0;` (sits above exit banner area if both visible — though both visible simultaneously is edge case only).

Actually: exit banner and paste-guard are mutually exclusive in normal operation (paste-guard fires while shell is running; exit banner fires when shell has exited). If both are somehow active, paste-guard takes `bottom: 28px` to clear the exit banner.

**When shell is running (no exit banner):** paste-guard sits at `bottom: 0`.

**Height:** Fixed 56px (two-line: content preview row + action row). When paste preview exceeds available horizontal width, it truncates with `…` — no multi-line expansion. This is a density-first choice.

**Background:** `--bg-3` (`#1f232e`)
**Border:** 1px solid `--accent-magenta` (`#c084d4`) on top edge (accent-magenta is the paste-guard's identity accent per color contract above)
**Border-top-left-radius / border-top-right-radius:** 0 (no rounding)

**Layout:**

**Row 1 — Preview (28px):**
```
[8px] ⏵ [4px] <paste preview, single line, truncated> [8px]
```
| Slot | Content | Color |
|------|---------|-------|
| Glyph `⏵` | U+23F5 | `--accent-magenta` |
| Preview text | First line of clipboard content, truncated at available width with `…` | `--fg-1` |
| Line count badge | If multi-line: `(N lines)` right-aligned | `--fg-3` 11px |

**Row 2 — Actions (28px):**
```
[8px] Send (⏎) [8px] Discard (Esc) ···spacer··· ⌘⇧V to paste without confirm [8px]
```
| Slot | Content | Color |
|------|---------|-------|
| Send button | `Send` with keyboard hint `⏎` | `--accent-blue` on `--bg-2` background |
| Discard button | `Discard` with keyboard hint `Esc` | `--fg-2` on `--bg-1` background |
| Right hint | `⌘⇧V skips this` | `--fg-3` 11px italic — informational only, not a button |

**Send button:** same styling as Restart button — `--bg-2` background, `--accent-blue` text, 1px `--border`, no radius.
**Discard button:** `--bg-1` background, `--fg-2` text, 1px `--border`, no radius.

**Keyboard interactions:**
- `Enter` → executes send: writes the pending paste to the PTY (with bracketed-paste wrapping if shell supports it), dismisses banner.
- `Escape` → discards: clears pending paste, dismisses banner. Nothing sent to PTY.
- `⌘⇧V` → bypass: sets bypass flag, fires normal paste without banner (banner never shows for this keypress).
- Any other key typed while banner is visible → **forwarded to terminal** — banner does not trap keyboard focus (non-modal).

**Dismiss animation:** None. Banner appears and disappears with no transition. Density-first means instant state change.

**Truncation of preview text:** Preview row shows first line of clipboard content only. If first line > ~80 chars (dynamic — fills available width minus glyphs and padding), truncate with `…`. The `(N lines)` badge shows the total line count from clipboard.

### 6. Focus / Blur State Summary

This is the complete and exhaustive focus specification. No other focus indicators exist.

| State | pane container `background` | pane container `box-shadow` | Border |
|-------|-----------------------------|-----------------------------|--------|
| Unfocused | `--bg-1` (`#11131a`) | none | 1px `--border` (`#262b38`) |
| Focused | `--bg-2` (`#171a23`) | `inset 0 0 0 2px rgba(90,124,255,0.18)` | 1px `--border` (unchanged) |

**What does NOT change on focus:**
- Border color (stays `--border` in both states)
- Header color (always `--bg-3`)
- Any text colors in header
- Any terminal content appearance

**What DOES change on focus:**
- Container background: `--bg-1` → `--bg-2` (subtle warmth)
- Container inset shadow: none → `inset 0 0 0 2px rgba(90,124,255,0.18)`

The xterm cursor blinks always (not conditional on pane focus in A2 — single pane, always focused). In A3 multi-pane: blinking cursor only in focused pane (A3 spec concern).

### 7. Initial / Empty State (pre-first-paint)

**What the user sees immediately on pane open:**
- Pane container with focused appearance (`--bg-2` + inset shadow) since A2 has one pane, always focused.
- Pane header: `●` dot in `--fg-3` (loading color), index `1`, cwd = current directory basename, shell name from `$SHELL` basename, process slot empty.
- Terminal body: shows the xterm canvas with `--bg-0` background. No placeholder text, no spinner, no skeleton state.
- Shell auto-launches per Q2 (auto-`$SHELL`) — in practice the terminal starts painting the shell prompt within ~100ms. The "loading" dot state is a brief flash before first PTY output.

**Status dot lifecycle:**
- Loading (pre-first-output): `--fg-3` (dim)
- Shell running / prompt active: `--accent-green`
- Foreground process running: `--accent-green` (same — shell is still alive)
- Shell exited: `--accent-red` (triggers exit banner)

**No empty-state illustration, no onboarding copy.** The terminal body is intentionally blank-canvas — the shell prompt is the empty state content.

### 8. Find Bar (PTY-03 — ⌘F Scrollback Search)

**Trigger:** `⌘F` while pane is focused.

**Placement:** `position: absolute; top: 22px; right: 0;` — floats below the header, overlapping the top of the terminal body. Does not push terminal content down.

**Width:** 280px fixed (sufficient for a search query + prev/next buttons).
**Height:** 32px.
**Background:** `--bg-3`
**Border:** 1px solid `--border-bright`; border-bottom only (butts against terminal canvas)
**Border-left:** 1px solid `--border-bright`
**Border-radius:** 0

**Layout (left to right):**
```
[8px] [Search input] [↑] [↓] [✕] [8px]
```

| Slot | Content | Color |
|------|---------|-------|
| Input | Placeholder `Find…` | `--fg-3` placeholder; `--fg-0` typed text |
| Prev match `↑` | Unicode `↑` (U+2191) | `--fg-2`; hover `--fg-0` |
| Next match `↓` | Unicode `↓` (U+2193) | `--fg-2`; hover `--fg-0` |
| Close `✕` | Unicode `✕` (U+2715) | `--fg-2`; hover `--accent-red` |

Input background: `--bg-2`. No border on input field itself — the bar container provides the border.

Match highlighting: handled by `@xterm/addon-search` `SearchAddon.findNext` / `findPrevious` with `decorations` option enabled. Match decoration colors: `background: rgba(90,124,255,0.35)` (current match); `background: rgba(90,124,255,0.15)` (other matches) — derived from `--focus`.

**Dismiss:** `Escape` or click `✕` closes find bar. Focus returns to terminal body.
**⌘⇧K — Clear scrollback:** Calls `Terminal.clear()`. No confirmation. No animation. Instant. Header does not change.

### 10. Accessibility

**Focus order:** A2 has a single pane. Tab stops within the pane:
1. Terminal body (xterm `canvas` element — receives focus on click)
2. When find bar is open: find input → prev → next → close (standard tab order)
3. When exit banner visible: Restart button is in DOM tab order
4. When paste-guard banner visible: Send → Discard (tab order); terminal body also focusable but banner buttons are prioritized

**Keyboard reachability of banner buttons:**
- Exit banner Restart button: reachable via Tab when banner is visible. `Enter` on focused button fires restart.
- Paste-guard Send/Discard: reachable via Tab. `Enter` on focused Send fires send. `Enter` on focused Discard fires discard.
- Paste-guard also has keyboard shortcuts (Enter/Escape) that do not require tabbing to the buttons.

**Contrast check against themes/default.css tokens (WCAG AA: 4.5:1 for normal text, 3:1 for large text):**

| Text | on Background | Contrast | WCAG AA |
|------|---------------|----------|---------|
| `--fg-0` (#e8eaf0) on `--bg-0` (#0a0b0e) | terminal body | ~18.5:1 | PASS |
| `--fg-0` (#e8eaf0) on `--bg-3` (#1f232e) | header, banners | ~13.2:1 | PASS |
| `--fg-1` (#aab0c0) on `--bg-3` (#1f232e) | secondary header text | ~7.8:1 | PASS |
| `--fg-2` (#6a7080) on `--bg-3` (#1f232e) | dim header text | ~4.1:1 | FAIL normal text (close); PASS large text |
| `--accent-blue` (#7aa2ff) on `--bg-2` (#171a23) | Restart button | ~6.1:1 | PASS |
| `--accent-green` (#6fd28f) on `--bg-3` (#1f232e) | status dot | N/A (decorative, 12px symbol) | Decorative |
| `--fg-3` (#444a5a) on `--bg-3` (#1f232e) | separators, hints | ~1.9:1 | FAIL (intentional — separators are decorative, not informational) |

**Note on `--fg-2` contrast:** At 12px/500 (header labels), `--fg-2` on `--bg-3` is ~4.1:1 — just below the 4.5:1 WCAG AA normal-text threshold. This is a **deliberate density-first tradeoff**. The text at this level (pane index, shell name) is supplemental chrome, not critical information. The primary content (terminal output) meets contrast at 18.5:1. If contrast compliance is required in a future accessibility pass, raise `--fg-2` to approximately `#7a8090`.

**Note on non-blocking checker FLAGs (acknowledged, no behavior change):**
- *Dim 2 — glyph-only buttons (`⋯ ↑ ↓ ✕`) lack aria-label:* A2 deliberately defers ARIA (dev-tool v0, power-user audience; xterm canvas produces no semantic DOM). This is an accepted, intentional posture for A2 — not a defect. Screen-reader support is deferred to a future accessibility phase.
- *Dim 4 — 11/12/13px type scale:* The three sizes are differentiated by color and font-family per the locked density-first Variant B contract (Inter UI chrome at 11/12px vs JetBrains Mono terminal at 13px). Accepted as-is — this is the canonical sketch 001 contract, not an oversight.

**Terminal content:** xterm.js renders terminal content controlled by the running shell. Terminal content contrast is user-controlled and out of scope for this spec. The xterm theme above ensures the default white/bright text has adequate contrast on the dark background.

**No ARIA required for A2:** The terminal canvas is an imperative rendering surface. xterm.js does not produce semantic DOM for accessibility. This is accepted for v0 (developer tool, power-user audience). Screen reader support is deferred.

### 11. Motion Contract

**Density-first means minimal-to-no animation.** The following table is exhaustive:

| Element | Motion | Rationale |
|---------|--------|-----------|
| Pane focus (bg lift + inset shadow) | Instant — no transition | 0ms. Immediate feedback for click. Transition would add lag. |
| Exit banner appear | Instant — no transition | 0ms. Banner snaps into position. |
| Exit banner dismiss (Restart) | Instant — no transition | 0ms. Banner disappears, terminal resumes. |
| Paste-guard banner appear | Instant — no transition | 0ms. Density-first: no slide-up animation. |
| Paste-guard banner dismiss | Instant — no transition | 0ms. |
| Find bar open / close | Instant — no transition | 0ms. |
| xterm cursor blink | `animation: blink 1s steps(2) infinite` | From themes/default.css `.cursor` class. 1s blink via xterm's built-in `cursorBlink: true`. |
| Status dot streaming pulse | `animation: pulse-glow 1.4s ease-in-out infinite` | From themes/default.css `.streaming-indicator`. Applied to status dot only when in streaming-active state (L2 only; not used in A2 PTY shell). |

**The only animations in A2 are the xterm cursor blink (built into xterm.js).**
All banner state transitions are instantaneous. `transition: none` should be explicitly set on banner and pane container elements to prevent accidental animation from any upstream CSS reset.
