Close Phase 4 with end-to-end codegen verification: PRD examples parse, analyze, generate readable Python snapshots, execute hermetically, and match the existing raw Python examples.

<threat_model>
Threats:
- T-04-06-A: Generated examples pass unit tests but diverge semantically from Phase 1 raw Python targets.
- T-04-06-B: Snapshot tests bless unreadable or minified output.
- T-04-06-C: Example execution pulls live providers, downloads embedding models, or leaves repo-local `.voss-cache` artifacts.
- T-04-06-D: Generated source imports undeclared modules or compiler internals.

Mitigations:
- Tests execute generated modules under `StubProvider` and compare outputs to `examples/raw_python/*`.
- Snapshot assertions include concrete grep-style readability checks, not only full-file equality.
- Support semantic routing uses fake Phase 3 manifests with embeddings; tests monkeypatch runtime encoding only if generated code would otherwise instantiate an encoder.
- Import allowlist checks parse generated AST imports and fail on `voss.*`, undeclared user modules, or unnecessary runtime names.
</threat_model>
