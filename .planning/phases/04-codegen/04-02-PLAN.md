<objective>
Implement the basic expression and statement/declaration lowering needed to produce parseable, readable async Python modules from simple Voss AST programs.
</objective>

<threat_model>
Threats:
- T-04-02-A: Expression emission drops parentheses and changes operator meaning.
- T-04-02-B: Top-level executable statements run on import instead of only under `__main__`.
- T-04-02-C: Codegen silently renames user identifiers and breaks `use` dependencies or stack traces.
- T-04-02-D: Unknown async behavior causes codegen to call generated Voss functions without `await`.

Mitigations:
- Unit tests cover nested `BinOp`/`UnaryOp` precedence and parse every expression inside a wrapper.
- Top-level executable statements are emitted only inside `async def main()` and guarded by `if __name__ == "__main__":`.
- `NameMangler` preserves spelling except `keyword.iskeyword`.
- Maintain a generated-function registry from `FnDecl` names and await calls to those functions when emitted in await-capable contexts.
</threat_model>

Extend `voss/codegen.py` with expression, type, and statement emitters.

Add `TypeEmitter` or private helpers:
- `string` -> `str`
- `int` -> `int`
- `float` -> `float`
- `bool` -> `bool`
- `list<T>` -> `list[T]`
- `dict<K,V>` -> `dict[K, V]`
- `probable<T>` -> `ProbableValue` and add runtime import `ProbableValue`
- unknown/named type -> mangled qualified final name.

Add `ProgramEmitter`:
- Collect generated function names from `FnDecl` before emission.
- Collect top-level declarations and top-level executable statements in source order.
- Add `asyncio` stdlib import when top-level executable statements exist.
- Emit declarations before `main`, preserving their original relative order.
- Emit top-level executable statements inside `async def main():`.
- Emit:
```python
if __name__ == "__main__":
    asyncio.run(main())
```

Add `ExpressionEmitter.emit(expr, *, await_context: bool = False) -> str`:
- `IntLit`, `FloatLit`, `StringLit`, `BoolLit`, `NullLit`, `Identifier`
- `BinOp`, `UnaryOp` with conservative parentheses for nested binary/unary operands
- `Call` with positional and named `Arg`
- When callee is a generated Voss function and `await_context=True`, prefix the call with `await `
- `Member`, `Index`, `ListLit`, `DictLit`, `Lambda`
- `SpawnExpr` -> `{AgentName}().spawn(args...)`

Add `StatementEmitter.emit(stmt)`:
- `ExprStmt`, `LetStmt`, `ReturnStmt`, `IfStmt`, `FnDecl`
- `FnDecl` always emits `async def name(params...) -> return_type:`
- Empty function or block emits `pass`.
- `ReturnStmt(Call(generated_fn))` in an async function emits `return await generated_fn(...)`.

Keep unsupported constructs explicit for now:
```python
raise CodegenError(f"unsupported AST node for codegen: {type(node).__name__}")
```
Do not silently emit `pass` for unsupported non-empty constructs.
