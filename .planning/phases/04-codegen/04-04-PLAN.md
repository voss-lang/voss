<objective>
Implement semantic routing codegen through Phase 3 program manifests and complete `use foo::bar` import lowering for GEN-04.
</objective>

<threat_model>
Threats:
- T-04-04-A: Generated match code recomputes embeddings at runtime or downloads sentence-transformers models.
- T-04-04-B: Codegen trusts unsafe manifest paths and reads outside the project/cache boundary.
- T-04-04-C: `use` lowering imports target modules during codegen and triggers user side effects.
- T-04-04-D: Similar-case labels drift from Phase 3 manifest labels, routing to the wrong arm.

Mitigations:
- Tests use a fake `.idx` manifest with embeddings and assert generated code contains `embeddings=` and no encoder/model builder calls.
- Manifest lookup resolves under `cache_dir` and reports `CodegenError` on missing/invalid `match_id`.
- `UseStmt` lowering is pure string emission from AST path parts.
- Match arm labels use manifest `case.label` values exactly and tests assert `case_0`, `case_1`, and wildcard behavior in emitted source.
</threat_model>

  <action>
Implement `UseStmt` and `MatchStmt` lowering in `voss/codegen.py`.

Use lowering:
- In the program prepass, for each `UseStmt`, call `ImportCollector.add_use(tuple(use.path), alias=use.alias)`.
- Do not import or resolve the module.
- If path length is less than 2, raise `CodegenError("use statement requires at least two path parts")`.

Manifest reader:
- Add private dataclass:
```python
@dataclass(frozen=True, slots=True)
class MatchIndex:
    match_id: str
    threshold: float
    cases: tuple[tuple[str, str], ...]      # (description, label)
    embeddings: tuple[tuple[float, ...], ...]
```
- Add `_load_match_index(match_id: str, *, source_path: str | Path | None, analysis: AnalysisResult | None, cache_dir: str | Path) -> MatchIndex`.
- Program stem is `Path(source_path).stem` or `"program"`.
- Manifest path is `Path(cache_dir) / f"{program_stem}.idx"` unless an `EmittedIndex.path` in `analysis.indexes` points to an existing path for the requested match; prefer the analyzer path when available.
- Read JSON with stdlib `json` inside codegen only; do not emit `json` import into generated source for this approach.
- Validate `version == 1`, `matches` is a list, and the requested `match_id` exists.
- Build `cases=((description, label), ...)` and `embeddings=((...), ...)` from manifest `cases` entries.
- If missing, raise `CodegenError(f"missing semantic match index for {match_id}")`.

Similar match codegen:
- For `MatchStmt` with at least one `SimilarPattern`:
  - Add runtime import `SemanticMatcher`.
  - Compute `match_id = f"match_{match.span.line_start}_{match.span.col_start}"`.
  - Emit stable matcher variable `_matcher_{n}` before the Python `match` statement:
```python
_matcher_1 = SemanticMatcher(
    cases=[("angry customer", "case_0"), ("refund request", "case_1")],
    threshold=0.8,
    embeddings=[[1.0, 0.0], [0.0, 1.0]],
)
match _matcher_1.match(userMessage):
    case "case_0":
        ...
    case "case_1":
        ...
    case _:
        ...
```
- Emit arm bodies from the corresponding `MatchCase.body`.
- Wildcard uses `case _:`.

Structural match fallback:
- If no `SimilarPattern` cases exist, emit native Python `match {scrutinee}:` and lower `ExprPattern` literals/expressions conservatively; wildcard remains `case _:`.
  </action>
