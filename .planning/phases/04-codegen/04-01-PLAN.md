<objective>
Establish the Phase 4 codegen contract gate and the smallest public codegen foundation: result metadata, writer, import collector, name mangling, package export, and initial readability/import tests.
</objective>

<threat_model>
Threats:
- T-04-01-A: Codegen begins against incomplete parser/analyzer contracts and silently invents AST shims.
- T-04-01-B: Generated Python imports compiler internals or undeclared user modules.
- T-04-01-C: Codegen executes user code while collecting imports or producing source.
- T-04-01-D: Unreadable minified output passes syntax tests but fails GEN-02.

Mitigations:
- Run the read-only `04-01-0` preflight before implementation; stop and report missing symbols/files if it fails.
- Import collection is data-only from AST nodes; no `importlib`, `__import__`, or user module inspection in `voss/codegen.py`.
- Unit tests assert no generated `from voss` / `import voss` compiler imports and stable sorted imports.
- `PythonWriter` tests assert four-space indentation, one final newline, and parseable multi-line blocks.
</threat_model>

  <action>
Create `voss/codegen.py`.

Target public API and dataclasses:
```python
from __future__ import annotations

from contextlib import contextmanager
from dataclasses import dataclass
from keyword import iskeyword
from pathlib import Path
from typing import Iterable

from .ast_nodes import Program, UseStmt
from .diagnostics import AnalysisResult

@dataclass(frozen=True, slots=True)
class CodegenResult:
    source: str
    imports: tuple[str, ...]
    requires_async_main: bool
    analysis: AnalysisResult | None = None

class CodegenError(Exception):
    pass

def generate_python(program: Program, *, source_path: str | Path | None = None, analysis: AnalysisResult | None = None, cache_dir: str | Path = ".voss-cache") -> CodegenResult: ...
```

If `analysis is None`, call `analyze(program, source_path=source_path, cache_dir=cache_dir)` imported lazily inside `generate_python`. If `analysis.ok` is false, raise `CodegenError("semantic analysis failed; refusing to generate Python")`. Warnings do not block.

Add `PythonWriter`:
- `lines: list[str]`
- `indent_level: int`
- `write(line: str = "") -> None` writes `"    " * indent_level + line` or a blank line.
- `blank() -> None` appends one blank line but never duplicates more than one blank line.
- `indent()` context manager increments/decrements indentation.
- `render() -> str` strips trailing blank lines and returns exactly one final newline.

Add `ImportCollector`:
- `stdlib: set[str]`
- `runtime: set[str]`
- `pydantic_base_model: bool`
- `user_imports: list[str]`
- `add_stdlib(name: str)`, `add_runtime(name: str)`, `add_base_model()`, `add_use(path: tuple[str, ...], alias: str | None = None)`.
- `add_use(("foo", "bar"))` records `from foo import bar`.
- `add_use(("foo", "bar", "baz"))` records `from foo.bar import baz`.
- Alias support emits `from foo import bar as alias`.
- `render() -> tuple[list[str], tuple[str, ...]]` returns import lines and import metadata. Runtime imports sort alphabetically inside `from voss_runtime import ...`.

Add `NameMangler`:
- `mangle(name: str) -> str`
- Return `name + "_"` only when `keyword.iskeyword(name)` is true; otherwise preserve spelling.

The initial `generate_python` may emit only imports plus `pass`/empty module for an empty `Program`, but it must return parseable Python and use the writer/import collector.
