<objective>
Implement lowering for runtime-backed language primitives: `probable<T>`, `ask`, `ctx`, `within/fallback`, memory declarations, and Voss `try/catch`.
</objective>

<threat_model>
Threats:
- T-04-03-A: Generated code uses sync `with ContextScope` or un-awaited runtime calls and fails at runtime.
- T-04-03-B: A global `ask(...)` compiles to an undefined function instead of a runtime context.
- T-04-03-C: `within/fallback` swallows non-budget exceptions or fails to route `BudgetExceededError` to fallback.
- T-04-03-D: Memory declarations execute external Chroma/source ingestion during codegen.

Mitigations:
- Tests grep exact async strings: `async with ContextScope`, `await ctx.add`, `await ctx.ask`, `await run_with_budget`.
- Global ask lowering is explicit and tested with `token_budget=4000`.
- Generated `within` catches only `BudgetExceededError`; other exceptions continue to propagate unless Voss `try/catch` catches them.
- Memory codegen emits constructor calls as source only; codegen never instantiates runtime memory objects.
</threat_model>

  <action>
Extend runtime construct lowering in `voss/codegen.py`.

Ctx state:
- Track `StatementEmitter.current_ctx_name: str | None`.
- Generate stable ctx names: `ctx`, `ctx_2`, etc. if nested.
- Inside explicit `CtxBlock`, `ask(...)` lowers to `await {ctx_name}.ask(...)`.
- `IncludeStmt(expr)` lowers to `await {ctx_name}.add(expr)`.
- `YieldStmt(expr)` inside ctx lowers to `return {expr}`; if the expression is `ask(...)`, emit `return await {ctx_name}.ask(...)`.

Global ask:
- When `Call(Identifier("ask"), ...)` is emitted in await-capable context and `current_ctx_name is None`, emit an implicit block at the statement level for assignments/returns:
```python
async with ContextScope(token_budget=4000) as ctx:
    intent: ProbableValue = await ctx.ask("Classify", return_type=ProbableValue)
```
- For plain `return ask("x")`, emit:
```python
async with ContextScope(token_budget=4000) as ctx:
    return await ctx.ask("x")
```
- For `let x: probable<T> = ask(...)`, pass `return_type=ProbableValue` and add runtime import `ProbableValue`.

Probable/confidence:
- Type annotations for `probable<T>` use `ProbableValue`.
- `ConfidenceGate(target=Identifier("intent"), op=">=", threshold=0.80)` in an `IfStmt` emits `if intent.confidence >= 0.8:`.
- Do not rewrite all `intent.value` usages; analyzer owns safety. Emit member access as normal Python.

Within/fallback:
- Normalize budget args:
  - `tokens` -> `token_limit`
  - `ms` -> `latency_ms`
  - `s` -> `latency_ms=value * 1000`
  - `usd` -> `cost_usd`
- Emit a nested async helper before the try block:
```python
async def _within_primary_1():
    ...
try:
    return await run_with_budget(_within_primary_1(), token_limit=5000, latency_ms=10000)
except BudgetExceededError:
    ...
```
- For non-returning primary blocks, assign result through the helper only when needed; keep simplest shape that passes tests.

Try/catch:
- `TryCatch(exc_name="err")` -> `try:` / `except Exception as err:`.
- `TryCatch(exc_name=None)` -> `except Exception:`.

Memory declarations:
- Detect `LetStmt` with `type_annot` `TypeRef` whose qualname is `memory.episodic`, `memory.semantic`, or `memory.working` and no initializer.
- Emit constructor calls:
  - `history = EpisodicMemory(capacity=20)`
  - `knowledge = SemanticMemory(source="./docs/", model="...")`
  - `notes = WorkingMemory()`
- Add corresponding runtime imports only when used.
