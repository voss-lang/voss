## Decisions Made

- Used middleware-level error handling for non-2xx responses, with wrapper-level guards for any raw `{ error }` result that still reaches a helper.
- Returned a composite SDK client object containing both the raw OpenAPI client and ergonomic methods; this lets focused helpers reuse auth/error behavior without duplicating fetch.
- Added local JSON-object aliases for some REST response shapes because the current generated contract emits those success bodies as generic JSON objects instead of named schemas.

## Deviations from Plan

- Some planned named response aliases (`SessionInfo`, `SavedSession`, `DoctorReport`, cost) are not emitted as named generated schemas in `types.ts`. The implementation uses generated response bodies where available and local JSON-object aliases where the OpenAPI snapshot is generic.
