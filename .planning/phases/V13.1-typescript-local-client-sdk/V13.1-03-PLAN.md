REST endpoints (all carry Bearer; app.py status codes verified):
- POST /session → 201 {id}    · GET /session → [SessionInfo]
- GET /sessions/saved (query cwd) · GET /session/:id → SessionInfo · DELETE /session/:id → 204
- POST /session/:id/message → 202 {status:"accepted"} (409 if a turn is running; 422 empty)
- POST /session/:id/abort → 202 · GET /session/:id/cost → cost rollup
- POST /session/:id/permission → 200 {v, status:"ok"|"stale"} · GET /doctor → DoctorReport
- 401 (constant-time bearer compare) on any request with a wrong/missing token.

Permission protocol (PROTOCOL §7 / app.py:252 PermissionReply, :380 reply_permission):
- Reply body: { v:1, id:<req_id>, choice } where choice ∈ "a" (allow once) | "A" (allow always) | "d" (deny); scope-expand uses "y" | "n".

Libs (D-02/D-03/D-04): `openapi-fetch` `createClient<paths>` + `client.use(middleware)` onRequest (RESEARCH Pattern 4). Native fetch. The generated `paths` type is in `src/generated/types.ts`. `VossApiError(status, detail)` from `src/errors.ts`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK client → loopback server | Untrusted-shaped responses cross here; bearer token must never be logged |
| caller → permission choice | Only the five protocol choices are valid; a wrong choice could mis-resolve a gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.1-06 | Information Disclosure | bearer token in logs/errors | mitigate | VossApiError carries only status + `{detail}`; never include the token in error messages or logs |
| T-V13.1-07 | Tampering | invalid permission choice mis-resolving a gate | mitigate | `PermissionChoice` is a closed TS union ("a"|"A"|"d"|"y"|"n"); compile-time rejects anything else |
| T-V13.1-08 | Spoofing | requests to a non-loopback base URL | accept | Base URL is supplied by the launcher (Plan 05) which binds `127.0.0.1`; loopback is the locked transport (PROTOCOL §1, no TLS by design) — note, not "fix" |
</threat_model>
