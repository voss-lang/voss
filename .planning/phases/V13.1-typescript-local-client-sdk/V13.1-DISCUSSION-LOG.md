## Codegen toolchain

| Option | Description | Selected |
|--------|-------------|----------|
| openapi-typescript + openapi-fetch | Types-only from openapi.json (incl. EventEnvelope via OpenAPI components) + typed native-fetch REST. Lightweight, isomorphic, no framework lock-in. | ✓ |
| orval | Types + client + react-query hooks generated. | |
| custom | Hand-rolled generator from EventEnvelope JSON schema. | |

**User's choice:** lock all (recommended)
**Notes:** `EventEnvelope` is already in OpenAPI components (`app.py:_force_event_schema`), so one openapi-typescript pass over `openapi.json` covers REST + the event union — no separate json-schema-to-ts step.

## HTTP + SSE libraries

| Option | Description | Selected |
|--------|-------------|----------|
| fetch + eventsource-parser | Native fetch (REST via openapi-fetch); fetch streaming body + eventsource-parser for SSE framing. Isomorphic. | ✓ |
| native EventSource | Browser-native SSE. | |

**User's choice:** lock all (recommended)
**Notes:** native `EventSource` rejected — can't send `Authorization: Bearer`, absent in Node. `eventsource-parser` is tiny + isomorphic.

## Package + module layout

| Option | Description | Selected |
|--------|-------------|----------|
| @vosslang/sdk @ sdk/typescript/, ESM-only, exports split, tsup+vitest | Sibling to @vosslang/cli; `"."`=browser core, `"./node"`=launcher. | ✓ |
| dual ESM+CJS / packages/ location / under npm/ | Alternatives offered. | |

**User's choice:** lock all (recommended); repo location `sdk/typescript/` (default accepted)
**Notes:** ESM-only; exports map enforces the browser-core / Node-launcher split.

## Codegen + CI wiring

| Option | Description | Selected |
|--------|-------------|----------|
| Checked-in generated types + CI regen diff gate | Commit generated types; CI regenerates and fails on non-empty git diff; reads V13's contracts/openapi.json; build hard-deps on V13. | ✓ |
| build-time-only generation | Generate during build, don't commit. | |

**User's choice:** lock all (recommended)
**Notes:** satisfies VSDK-TS-01 byte-identical acceptance; catches TS-side protocol drift.
