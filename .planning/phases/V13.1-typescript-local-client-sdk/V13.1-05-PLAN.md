`voss serve` boot (serve.py — verified):
- Registered command `voss serve` (cli.py:3810 serve_cmd; in AGENT/main group at cli.py:4253). Spawn entry: `python -m voss.cli serve`.
- serve.py:48-53 — `sock.bind((host,0))` then `sock.listen(128)` BEFORE the handshake print (race-free, Pitfall 5 — no sleep/retry needed).
- serve.py:59 — prints EXACTLY one line: `{"v":1,"port":<N>,"token":"<url-safe-32>"}` then flush.
- serve.py:64-65 — `_watch_stdin_eof` daemon starts ONLY if `not sys.stdin.isatty()`. So stdin MUST be piped (Pitfall 7) or the server never self-terminates on dispose → orphan.
- serve.py:22-26 — getppid() poll (2s) is the secondary orphan guard.

Spawn pattern (RESEARCH Pattern 6, mirrors tests/harness/mcp/test_mcp_serve_e2e.py:38-51 which uses `sys.executable -m voss.cli ...`, stdin/stdout PIPE, stderr inherit):
- `spawn(python, ["-m","voss.cli","serve"], { cwd, stdio:["pipe","pipe","inherit"], env })`.
- python bin: accept a `python`/`pythonBin` option; default `process.env.PYTHON_BIN ?? "python3"` (RESEARCH Open Q3). NOTE: the repo `.venv/bin/python` is the tested interpreter — the integration suite (Plan 06) passes PYTHON_BIN accordingly.
- Read handshake: `readline.createInterface({ input: proc.stdout }).once("line", ...)` → `JSON.parse`. Reject on proc `error`/early `exit`; 10s timeout.
- Dispose: `proc.stdin?.end()` (EOF → server _watch_stdin_eof fires) + `proc.kill("SIGTERM")`; register `process.on("exit", dispose)` for kill-on-drop.

depcruise (Pattern 8): `.dependency-cruiser.cjs` forbidden `core-no-node-builtins`, from `^dist/index\.js$`, to `dependencyTypes:["core"]`. Run `depcruise --config .dependency-cruiser.cjs dist/index.js` (the `check:core-imports` script). It exits non-zero if ANY module reachable from dist/index.js imports a Node builtin (incl. transitive); it does NOT scan dist/node.js.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| launcher → spawned `voss serve` process | The launcher chooses the binary/args/cwd that becomes a running server |
| core entrypoint → browser bundle | The `.` export must be safe to bundle for a browser (no Node builtins) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.1-12 | Elevation of Privilege | attacker-controlled python bin / serve path | mitigate | Spawn args are fixed (`-m voss.cli serve`); only the python interpreter is configurable via an explicit `python`/`PYTHON_BIN` option (caller-controlled, documented) — no shell string interpolation (use array args, not `shell:true`) |
| T-V13.1-13 | Denial of Service | orphan `voss serve` on host crash | mitigate | stdin piped + EOF on dispose + SIGTERM + `process.on("exit")` kill-on-drop; server getppid-poll is the server-side backstop (PROTOCOL §3) |
| T-V13.1-14 | Information Disclosure | handshake token leaking to logs | mitigate | Token is read from the child's stdout handshake into memory only; never written to stderr/stdout/logs by the launcher |
| T-V13.1-06b | Tampering | node:* import leaking into the browser-safe core | mitigate | dependency-cruiser `core-no-node-builtins` rule fails the build on any reachable builtin (VSDK-TS-06) |
</threat_model>
