## Accomplishments

- Added `VossLauncher.start({ python?, cwd? })`, which spawns `python -m voss.cli serve` with piped stdin/stdout, parses the `{ v, port, token }` handshake, and returns `{ client, pid }`.
- Added `VossLauncher.dispose()`, process-exit cleanup, and failed-start cleanup so the child process receives stdin EOF plus `SIGTERM`.
- Replaced `src/node.ts` with a thin `./node` entrypoint exporting launcher APIs only.
- Verified dual-entrypoint build artifacts and proved the core `dist/index.js` graph has no reachable Node builtins.
