<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed snapshot → downstream codegen | The snapshot is the source for all SDK types (V13.1–.4); a wrong/stale snapshot silently propagates to every client |
| exporter token → emitted schema | A randomly-generated bearer token could leak into the OpenAPI output and break byte-stability |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.1-01 | Tampering | contracts/*.json drift undetected | mitigate | Drift gate (Task 2) re-exports and byte-compares; negative test proves it fails on change |
| T-V13.1-02 | Information Disclosure | bearer token leaking into committed openapi.json | mitigate | Use `FIXED_TOKEN` constant, not `secrets.token_urlsafe`; no security scheme is registered in app.py so the token does not enter the schema — fixed value is defensive and byte-stable |
| T-V13.1-03 | Repudiation/Drift | exporter and gate using different dump formats | mitigate | Both Task 1 and Task 2 pin identical `indent=2, sort_keys=True` + trailing newline + same FIXED_TOKEN |
</threat_model>
