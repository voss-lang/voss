<decisions>
## Implementation Decisions

### Codegen toolchain
- **D-01:** Types generated with `openapi-typescript` over V13's committed `contracts/openapi.json`. Because `app.py:_force_event_schema` forces `EventEnvelope` into OpenAPI components, this single pass captures BOTH the REST request/response shapes AND the full `AgentEvent` discriminated union — no separate JSON-schema-to-TS step.
- **D-02:** Typed REST calls use `openapi-fetch` (pairs with openapi-typescript, native-fetch wrapper, ~6kb). No `orval` / no react-query — framework-agnostic, isomorphic, zero UI-framework lock-in.

### HTTP + SSE libraries
- **D-03:** REST transport = native `fetch` via `openapi-fetch`. Works in Node 18+ and browsers.
- **D-04:** SSE transport = native `fetch` (streaming body) + `eventsource-parser` for SSE framing. Native `EventSource` is REJECTED: it cannot send the required `Authorization: Bearer` header and is absent in Node. `eventsource-parser` is tiny and isomorphic. Each parsed `data` is validated/narrowed against its `type` to yield the typed `AgentEvent`.

### Package + module layout
- **D-05:** Package name `@vosslang/sdk` (sibling to the existing `@vosslang/cli` M6 wrapper — NOT an extension of it).
- **D-06:** Repo location `sdk/typescript/`.
- **D-07:** ESM-only.
- **D-08:** `exports` map enforces the isomorphic split: `"."` → browser-safe core (REST + SSE + permission helpers + types, zero `node:*`); `"./node"` → Node-only launcher/supervisor.
- **D-09:** Build tool `tsup` (emits JS + `.d.ts`); test runner `vitest`.

### Codegen + CI wiring
- **D-10:** Generated types are CHECKED IN (committed), not build-time-only — keeps the package buildable/inspectable without re-running codegen and enables the byte-identical drift check.
- **D-11:** Codegen reads `contracts/openapi.json` from the V13 snapshot (the exact committed path is finalized by V13's discuss; V13.1 consumes whatever path V13 locks).
- **D-12:** CI regenerates types and fails if `git diff` on the generated file is non-empty — satisfies VSDK-TS-01's "regenerate byte-identically" acceptance and catches protocol drift on the TS side.
- **D-13:** Build hard-depends on V13 artifacts existing. V13.1 may be planned ahead, but execution blocks until V13 ships `contracts/openapi.json` + the event-union schema.

### Claude's Discretion
- Exact `tsup`/`vitest` config, lint rule mechanism for the zero-`node:*` core check (D-08 / VSDK-TS-06), test fixture wiring for `VOSS_SERVE_FAKE_TURN`, and the public TS API ergonomics (method names, error class shape) are left to research/planning within the SPEC's acceptance criteria.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `voss/harness/server/app.py:_force_event_schema` — already publishes `EventEnvelope` in OpenAPI components, so one `openapi-typescript` pass yields the event union types. No extra schema plumbing needed.
- `VOSS_SERVE_FAKE_TURN` env seam (app.py `_run_turn`) — emits a canned turn over the real event/SSE path with no provider creds. The integration-test driver for CI.
- `npm/` (`@vosslang/cli`) — sibling package shape reference (npm org `@vosslang`, package.json conventions), NOT a base to extend.

### Established Patterns
- Handshake-then-bind in `serve.py`: client spawns `voss serve` with stdin PIPED (so EOF supervision fires), reads exactly one stdout JSON line `{v,port,token}`, then `http://127.0.0.1:<port>` + `Authorization: Bearer <token>` on every request incl. the SSE GET. The Node launcher mirrors this.
- SSE framing per PROTOCOL §6: `event: <type>` / `data: {"v":1,...}` / `id: <seq>`; first event `server.connected`, turn end `session.idle`; client disconnect cancels the in-flight turn (§8).
- PROTOCOL mirrors OpenCode `packages/sdk/js` shapes — an OpenCode-style client degrades gracefully; useful prior art for client ergonomics.

### Integration Points
- REST endpoints (app.py): `POST/GET /session`, `GET /sessions/saved`, `GET/DELETE /session/:id`, `POST /session/:id/message`, `POST /session/:id/abort`, `GET /session/:id/cost`, `POST /session/:id/permission`, `GET /session/:id/events` (SSE), `GET /doctor`.
- Permission round-trip (PROTOCOL §7): `permission.updated {id,tool_name,args,dimension}` → reply `POST /session/:id/permission {v,id,choice}`, choice ∈ `a|A|d` (+ `y|n` scope).

</code_context>
