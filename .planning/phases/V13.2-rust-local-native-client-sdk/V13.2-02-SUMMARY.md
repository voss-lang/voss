## Accomplishments

- Added `VossClient` with bearer-authenticated methods for create/resume/delete session, post message, abort, permission reply, saved sessions, cost, and doctor.
- Added non-2xx mapping to `VossError::HttpStatus { status, detail }`, extracting the server `{detail}` field when present.
- Added a hand-written `Debug` implementation that prints `token: "[REDACTED]"` and never exposes the bearer token literal.
- Added `Handshake::from_line` and `Handshake::into_client` for the `voss serve` `{v,port,token}` startup line.

## Decisions Made

- `VossClient::auth` is `pub(crate)` so the future SSE stream module can reuse the same bearer-token attachment path.
- `VossClient` keeps `inner`, `base`, and `token` crate-visible for planned internal SDK modules while keeping the public Debug output token-safe.
- No new dev dependencies were added; the `ok_or_detail` behavior test covers the same detail-extraction logic through `http_status_error`.

## Deviations from Plan

- None requiring scope change. The ok-or-detail test uses a pure helper instead of constructing a `reqwest::Response`, which the plan allowed as the fallback path.

## Issues Encountered

- The initial `cargo test -p voss-sdk --lib -- handshake` filter only selected two tests because two auth tests did not include `handshake` in their names. Those tests were renamed so the exact filtered command now runs all four handshake behavior cases.
