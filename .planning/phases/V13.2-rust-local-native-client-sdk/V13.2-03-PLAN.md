## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE server → SDK stream | Untrusted event frames deserialized into the typed union; malformed data → typed Decode error, never a panic |
| SDK stream → server (disconnect) | Dropping the stream signals turn cancellation via TCP close (PROTOCOL §8) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.2-12 | Denial of Service | `event_stream` frame loop | mitigate | A malformed frame yields `Err(VossError::Decode)` and the stream continues/terminates cleanly — never panics or hangs (no `.unwrap()` on frame data). |
| T-V13.2-09 | Information Disclosure | SSE GET bearer auth | mitigate | Bearer attached via the same `VossClient` auth path; token not interpolated into any yielded error string. |
| T-V13.2-13 | Denial of Service | dropped stream / leaked task | mitigate | unfold owns the reqwest Response in its state; dropping the Stream drops the Response (TCP close) with no spawned background task to leak — verified by Wave 4 `sse_drop_midstream`. |
</threat_model>
