## Codegen tool

| Option | Description | Selected |
|--------|-------------|----------|
| typify (types only) + hand-written client | Oxide typify: JSON Schema → serde types (tagged enum + REST bodies); thin REST/SSE client hand-written on voss-tui net.rs | ✓ |
| progenitor (full client + types) | Oxide progenitor: complete reqwest client + types from openapi.json; SSE+supervisor still hand-written | |
| Hand-roll serde from the schema | No codegen crate; manual structs guarded by a match test | |

**User's choice:** typify (types only) + hand-written client
**Notes:** SSE isn't OpenAPI-modeled so a full client generator buys little; the loopback+bearer+supervisor+SSE shape doesn't match progenitor's generated client. Hand-roll rejected by SPEC's "generated, not hand-written" constraint.
