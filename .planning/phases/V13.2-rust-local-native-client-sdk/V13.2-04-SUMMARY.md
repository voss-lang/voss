## Accomplishments

- Added `Supervisor { client }` with a private `tokio::process::Child`.
- Added `python_path()` resolving `VOSS_PYTHON` then repo `.venv/bin/python` then `python3`.
- Added `spawn()` and `spawn_with(...)` that launch `python -m voss.cli serve --port 0` with piped stdin/stdout, inherited stderr, `kill_on_drop(true)`, and extra test env support.
- Added a 20s stdout handshake wait, parsing with `Handshake::from_line`, then returning a pre-bound `VossClient`.
- Added `Drop` and `shutdown()` cleanup paths, plus a bad-interpreter test proving typed error behavior instead of hang/panic.

## Decisions Made

- `Supervisor::shutdown(self)` consumes the supervisor, starts kill, and waits to reap; the existing Drop impl may attempt a second ignored kill after wait, which is harmless and keeps the belt-and-suspenders cleanup contract simple.
- The bad-interpreter test pattern-matches the `Result` instead of using `unwrap_err()`, avoiding a production `Debug` requirement on `Supervisor`.
