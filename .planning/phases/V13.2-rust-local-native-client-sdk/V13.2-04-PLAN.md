## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDK → OS child process | Supervisor owns a `voss serve` child; lifecycle (kill/reap) must not leak processes |
| SDK → interpreter path | Caller/env-supplied interpreter path is executed; a bad path must fail typed, not hang |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-V13.2-ORPHAN | Denial of Service | `Supervisor` child lifecycle | mitigate | Three-level cleanup: `Command::kill_on_drop(true)`, `impl Drop` start_kill, explicit `shutdown()` start_kill+wait. Wave 4 `supervisor_no_orphan` asserts no `voss serve` PID survives drop. kill_on_drop applied before any fallible op (Pitfall 4) so a mid-spawn panic cannot orphan. |
| T-V13.2-14 | Denial of Service | bad interpreter spawn | mitigate | `spawn_with` maps spawn failure to `VossError::Spawn`; the 20s handshake timeout bounds the wait so a non-server interpreter can't hang. Task 1 test asserts a typed error. |
| T-V13.2-09 | Information Disclosure | handshake token handling | mitigate | Token flows from handshake line directly into VossClient (token-redacting Debug from Plan 02); supervisor never logs the handshake line verbatim. |
</threat_model>
