## Decisions Made

- Added safe public accessors instead of parsing `Debug` output or exposing bearer tokens.
- Kept `auth_bad_token`, `rest_roundtrip`, `sse_event_sequence`, `sse_drop_midstream`, and `supervisor_no_orphan` hermetic with `VOSS_SERVE_FAKE_TURN=1`.
- Used the real-turn path for `post_while_busy`; this avoids a flaky 409 assertion against the fast fake-turn branch.
- Left the permission test skip-not-fail when no provider credentials or no permission gate are available; the operator confirmed the permission round-trip in a credentials environment.

## Deviations from Plan

- Added `VossClient::base_url()` and `Supervisor::pid()` to make the integration tests use stable public API instead of parsing debug output or process lists.
- `post_while_busy` uses the real-turn path with skip-not-fail semantics rather than FAKE_TURN, because FAKE_TURN finishes too quickly to reliably return 409.

## Issues Encountered

- Initial integration compile imported `VossError` from the crate root; fixed to `voss_sdk::error::VossError`.
- The first 409 test attempt used FAKE_TURN and saw the second post succeed because the fake turn had already completed.
- Test runs generated a `.voss/sessions/*.json` file under `crates/voss-sdk`; it was deleted and not committed.
