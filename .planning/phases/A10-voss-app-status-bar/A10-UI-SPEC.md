A10 MUST NOT redefine any token value from A1. All status bar chrome is expressed using
tokens already declared in `apps/voss-app/src/styles/variant-b.css`.

**Token system reference:** `variant-b.css` — `--bg-0..3`, `--fg-0..3`, `--border`, `--border-bright`, `--focus`, `--focus-glow`, `--accent-*`, `--font-mono`, `--titlebar-height`.

## Spacing Scale

A10 inherits the full A1 spacing scale without modification. Status-bar-specific
measurements are declared as named structural constants below.

| Token | Value | A10 Usage |
|-------|-------|----------|
| xs | 4px | Gap between glyph prefix and label text within a cluster; bell badge offset |
| sm | 8px | Horizontal padding inside each cluster; gap between clusters |
| md | 16px | Minimum horizontal padding from viewport edge to outer clusters |
| lg | 24px | Not used in status bar chrome |
| xl | 32px | Not used in status bar chrome |

**Exceptions:** none. The bar is a fixed 22px strip — all internal spacing follows xs/sm.

### Named Structural Constants for A10 Status Bar Chrome

| Constant | Value | Rationale |
|----------|-------|-----------|
| Status bar height | **22px** | BAR-05 requirement. Matches `--titlebar-height` (A1). |
| Status bar z-index | **10** | Above GridRoot (z:0), below popovers (z:20), well below overlays (z:50). |
| Cluster button min-width | **28px** | Minimum tap/click target; matches pane header menu button. |
| Popover z-index | **20** | Above status bar, below command palette / settings overlays (z:50). |
| Popover width (left) | **240px** | Recent projects list — wide enough for typical project names. |
| Popover width (center) | **280px** | Pane detail card — needs room for full cwd path. |
| Popover width (right/bell) | **320px** | Notification list — widest content; messages need breathing room. |
| Popover max-height | **320px** | Constrains notification and recent-projects lists to scroll beyond this. |
| Bell badge size | **16px** | Circle badge; sits at top-right of bell glyph; diameter is 4×4 grid. |
| Bell badge max count | **"99+"** | Truncate at 99 to keep badge within 16px circle. |
| Cog button width | **28px** | Square icon-only button. No label. |
| Notification row height | **40px** | 10×4 grid — accommodates message text (12px) + timestamp (11px) in two lines. |
| Recent project row height | **32px** | 8×4 grid — single-line project name at 12px. |
| Pane detail card row height | **24px** | 6×4 grid — label + value on same line, mono font. |

**Border radius:** 0px on all A10 chrome — Variant B absolute rule. No exceptions.

## Typography

Status bar is terminal-density chrome. Mono font throughout. Two sizes maximum.

| Role | Size | Weight | Line Height | Font | A10 Usage |
|------|------|--------|-------------|------|---------|
| Cluster label | 11px | 400 | 1.0 | `--font-mono` | Project name, branch name, pane cwd, pane count — all inline cluster text |
| Popover body | 12px | 400 | 1.4 | `--font-mono` | Notification messages, recent project names, pane detail values |
| Popover label | 11px | 400 | 1.0 | `--font-mono` | Pane detail card field labels (cwd:, shell:, pid:, index:, cmd:); notification timestamps |
| Popover heading | 11px | 500 | 1.0 | `--font-mono` | "Recent Projects" header, "Notifications" header, "Pane Detail" header |

**Size rationale:** 11px matches the titlebar title text (confirmed in `Titlebar.tsx` — `font-size: 11px`). This is the established "chrome bar text" size for Variant B. Popover body steps up to 12px (matching pane header and toast) for readability in a scrollable list.

| Token | Hex | Specific elements |
|-------|-----|-------------------|
| `--accent-green` | `#6fd28f` | Bell badge background when unread count > 0. |
| `--accent-blue` | `#7aa2ff` | Active cluster button left border indicator (2px) when popover is open. |

## Layout Contract

### Status Bar Structure

```
┌─────────────────────────────────────────────────────────────┐  22px
│ [◆ project-name · ⎇ main]   [▢ ~/cwd · zsh]   [▢3 · 🔔 · ⚙]│
│  ←── LEFT CLUSTER ──→         ←─ CENTER ─→     ←─ RIGHT ──→  │
└─────────────────────────────────────────────────────────────┘
  border-top: 1px solid --border (separates from GridRoot)
```

The status bar occupies the bottom row of `App.tsx`'s flex column, below `<GridRoot>`. It is `flex-shrink: 0`.

### Cluster Layout

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [LEFT]                [CENTER]               [RIGHT]        │
│  flex: 1               flex: 0 (shrink)       flex: 0        │
│  justify: flex-start   position: absolute     justify: flex-end│
│                        left: 50%              (right-anchor)  │
│                        transform: translateX(-50%)           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

| Cluster | Flex | Content | Click target |
|---------|------|---------|-------------|
| Left | `flex: 1; justify-content: flex-start` | `◆ {project}` · `⎇ {branch}` (branch hidden when no git — D-11) | Entire cluster — one click opens left popover |
| Center | Absolutely centered | `▢ {cwd}` · `{shell}` | Entire cluster — one click opens center popover |
| Right | `flex: 0; justify-content: flex-end` | `▢{N}` · bell-button · cog-button | Bell button, cog button are separate click targets |

**Glyph inventory:**
- `◆` — project marker (filled diamond, U+25C6)
- `⎇` — branch (alternative key, U+2387) — hidden when no git repo (D-11)
- `▢` — pane/terminal indicator (white square, U+25A2)
- `🔔` — bell (U+1F514) — acceptable Unicode fallback; use plain text `○` if glyph renders incorrectly
- `⚙` — cog (U+2699) — dispatches `open-settings` command (A9 D-02)
- `·` — separator dot (U+00B7) in `--fg-3`

**No-project state (BAR-08):** Left cluster shows `◆ no project · ⌘O to open` in `--fg-2`. No branch segment.

### Popover Positioning

All popovers open **above** the status bar (upward direction), flush with the triggering cluster's horizontal edge. Edge-clamp: if popover would overflow viewport right/left edge, shift horizontally until fully visible. Never overflow viewport.

```
┌───────────────────────┐
│  Popover content       │  ← opens upward above bar
│  background: --bg-3    │
│  border: 1px --border  │
└───────────────────────┘
             ↑
═══════════════════════════  ← status bar
```

No arrow/caret. No drop-shadow (Variant B has no shadows — use border only).

## Notification Store Schema

```typescript
interface NotificationEntry {
  id: number;           // autoincrement
  severity: 'success' | 'warning' | 'error' | 'info';
  message: string;      // human-readable, max 120 chars
  source: string;       // e.g. "pane-exit", "layout-save", "settings-reload", "update", "app-error"
  timestamp: number;    // Unix ms (Date.now())
  read: boolean;        // false until bell popover opened
}
```

Store size: last 50 entries in memory, last 50 written to `~/.config/voss-app/notifications.json` on quit (D-07). `addNotification()` appends to store AND calls `showToast()` (D-05 unified stream).

## Git Branch Display Contract (D-09..D-11)

| State | Left cluster display |
|-------|---------------------|
| Git repo, branch known | `◆ {project} · ⎇ {branch}` |
| Git repo, detached HEAD | `◆ {project} · ⎇ (detached)` in `--fg-3` |
| No git repo | `◆ {project}` (no ⎇ segment at all — D-11) |
| No project open | `◆ no project · ⌘O to open` in `--fg-2` |

Branch text: max 32ch before `text-overflow: ellipsis`. Feature branches like `feature/my-long-branch-name` truncate to `feature/my-long-branch-n…`.

Rust watches `.git/HEAD` (D-10). On change event, Tauri emits `branch-changed` event to frontend. Frontend listener updates branch signal. Debounce: 200ms after last event before re-reading (prevents thrash during rebase).

## Bell Badge Contract (D-05, D-06)

| State | Badge display |
|-------|--------------|
| No unread notifications | Badge hidden entirely |
| 1–9 unread | Badge visible, shows count number, background = severity color of highest-severity unread |
| 10–99 unread | Badge shows count number |
| 100+ unread | Badge shows "99+" |
| Bell popover opens | Badge immediately hidden; all entries marked read |

Badge position: top-right corner of bell glyph, 16px diameter circle, 10px text, `--bg-0` text color (on colored badge background for contrast).

## Interaction Contracts

### Popover Behavior (D-01, D-04)

| Trigger | Action |
|---------|--------|
| Click cluster / bell button | Open that cluster's popover; close any other open popover |
| Click outside popover | Dismiss popover |
| `Esc` key | Dismiss active popover; return focus to previously-focused pane |
| Open second popover while first open | Close first, open second (one at a time — D-01) |
| Click cluster button when popover already open | Close popover (toggle behavior) |
| Cog button click | Dispatch `open-settings` command (no popover for cog) |

### Recent Projects Row Click (D-02)

| Action | Result |
|--------|--------|
| Click project row | Switch active project; close popover; status bar left cluster updates |

### Bell Popover Open (D-06)

| Action | Result |
|--------|--------|
| Bell popover opens | Unread badge resets to 0 immediately; all entries marked `read: true` |
| Bell popover closes | No state change |

### Focus Management

| Event | Focus behavior |
|-------|---------------|
| Popover opens | Popover receives focus (keyboard-navigable list) |
| Esc / click-outside | Focus returns to GridRoot focused pane (the pane that was active before) |
| Tab within notifications list | Cycle through row items |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard access | All cluster buttons and popover rows reachable by Tab; Enter activates; Esc closes |
| Focus visible | Cluster buttons show `inset 0 0 0 1px var(--focus)` on keyboard focus |
| Screen reader | `role="toolbar"` on StatusBar, `aria-label` on each cluster button describing current state (e.g. `aria-label="Project: voss, Branch: main. Open recent projects"`) |
| Bell badge | `aria-label` on bell button includes unread count: `"Notifications, 3 unread"` |
| Notification list | `role="list"` on list, `role="listitem"` on each row, `aria-live="polite"` on notifications popover |
| Recent projects | `role="list"` on list; active project has `aria-current="true"` |
| Reduced motion | No animations defined for status bar chrome. Popover appearance is instant (no fade-in). |
