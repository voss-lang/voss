**Hybrid bootstrap rationale (key decisions, locked):**
1. **Q1 hybrid B** — harness owns the 3 yml stubs + project.json deterministically; LLM owns only architecture.md. Schema-violation risk goes from 5 fs_writes × strict-pydantic → 0 (yml stubs round-trip through pydantic during construction; project.json built from validated `ProjectMeta` instance).
2. **Q2 prose-only B** — harness pre-computes sha, file_count, dir-tree, primary_language, manifest head, README head; injects into prompt. LLM emits 1 fs_write. Reduces /analyze from 5-10 tool calls to 1.
3. **Q3 preserve-if-exists B** — `.voss/{project.json, *.yml}` only-write-if-absent. `.voss/architecture.md` always overwritten (refresh semantics). `.voss-cache/repo.idx` always rebuilt. User edits to project.json/yml survive /analyze re-runs.
4. **Q4 surgical helpers C** — extract `_handle_analyze(state)` + `_handle_save_plan(state, line)` from `_run_repl`; existing slash branches stay inline.
5. **Q5 no agent.py edit A** — `TurnResult.plan` already exists at agent.py:111 and is populated at both return sites (agent.py:204, 276). cli.py just reads `result.plan` after each turn.
6. **Q6 heuristic project.json α** — `name = cwd.name`; `primary_language` from extension histogram; `type = "library"`, `entry_points = []` defaults. User edits manually if wrong (preserve-if-exists protects edit).
7. **Q7 inject manifest+README B** — bootstrap prompt carries manifest (pyproject.toml | package.json | Cargo.toml | Package.swift | go.mod | Gemfile, whichever exist) + README.md head, each capped at 4k chars.
8. **Q8 manual /save-plan + kwargs-style α/iii** — manual trigger only; steps render as `- <tool>(<k>=<json_v>, ...) — <why>`; frontmatter adds `model` field.
