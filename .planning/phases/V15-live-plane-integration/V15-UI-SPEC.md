# Phase V15 — UI Design Contract: Live Plane Integration

> Visual and interaction contract for the Structured Protocol Pane and supporting
> affordances. Generated by gsd-ui-researcher from mockup + existing tokens.
> Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — CSS custom properties only |
| Preset | not applicable |
| Component library | none (custom Solid components, no shadcn, no Radix) |
| Icon library | none (Unicode glyphs and CSS shapes only, matching existing pane chrome) |
| Font — mono | `var(--font-mono)` → JetBrains Mono, SF Mono, Menlo, ui-monospace |
| Font — UI | `var(--font-ui)` → Inter, -apple-system, SF Pro Text, system-ui |
| Font — display | `var(--font-display)` → Poppins, system-ui (section labels only) |
| Token source | `apps/voss-app/src/styles/variant-b.css` + Voss Ignite theme |

**No new npm or cargo dependencies.** All new visual elements use the existing
token vocabulary (`--bg-*`, `--fg-*`, `--accent-*`, `--focus`, `--border*`,
`--role-*`, `--focus-soft`, `--tool`). No raw hex values in component code.

**Variant B radius rule:** `border-radius: 0` everywhere except traffic-light
circles (50%) and scrollbar thumbs (4px). This rule extends to all new V15
components.

---

## Spacing Scale

V15 reuses the existing 4-point scale already established in the codebase.
No new spacing tokens.

| Value | Existing Usage | V15 Usage |
|-------|---------------|-----------|
| 4px | Icon gaps, inline padding (`xs`) | Transcript row vertical padding; tool-line collapse chevron margin; statusbar indicator gap |
| 8px | Compact element spacing (`sm`) | Tool-line collapsed row height padding; boot-placeholder icon gap; permission-gate label and button gaps |
| 12px | Section padding (cockpit) | Transcript body horizontal padding; session-list row padding |
| 16px | Default element spacing (`md`) | Permission-gate block margin top/bottom |
| 24px | Section breaks | Boot-placeholder vertical centering padding |

Exceptions:
- Permission gate border-radius: 0 (Variant B rule)
- Transcript scroll area: `padding: 8px 12px` (8 vertical, 12 horizontal — matches existing `.cockpit-peek__body`)
- Session-list row height: 28px (matches `.org-run-picker__row`)
- `outline-offset` values (`-1px`, `2px`) are visual-alignment values for focus rings, exempt from the 4-point layout grid

## Typography

All type in V15 structured pane uses existing font families. New elements
follow a size floor of 11px for body and interactive text (A12 rule — 9px
mockup labels round up). **Explicit exception: 10px is sanctioned for
tertiary metadata only** — tool-row chevron, plan confidence badge, final-row
meta, generic fallback rows, and the permission-gate resolved label — keeping
metadata visually subordinate to the 11px body. No interactive element uses 10px.
Exactly two weights: 400 (body) and 600 (anchors, labels, primary actions).

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Transcript body | `--font-mono` | 11px | 400 | 1.62 | Tool lines, stream text, generic fallback rows |
| Task header | `--font-mono` | 11px | 600 | 1.4 | EM task header line (user task echo) — primary anchor |
| Section label | `--font-display` (Poppins) | 11px | 600 | 1 | "Server sessions" sidebar section header |
| Session meta | `--font-ui` | 11px | 400 | 1.4 | Session list id/title/age in sidebar |
| Permission gate label | `--font-ui` | 11px | 600 | 1.4 | Gate warning line |
| Permission gate args | `--font-mono` | 11px | 400 | 1.4 | Tool name + args line inside gate |
| Permission gate buttons | `--font-ui` | 11px | 400 | 1 | Deny / Allow once / Allow for scope |
| Boot placeholder | `--font-ui` | 12px | 400 | 1.4 | "Starting…" + elapsed counter |
| Ended banner | `--font-mono` | 12px | 400 | 1 | ExitBanner reuse — "[session ended]" |
| Statusbar live indicator | `--font-mono` | 11px | 400 | 1 | `● live · voss serve :<port>` |

| Border bright | `--border-bright` | `#353b4a` | Permission gate button border; session-list dividers |
| Accent (10%) — Focus / CTA | `--focus` | `#5a7cff` | Attach button; Retry start button in spawn-failure state; focus rings |
| Accent — Live stream | `--accent-cyan` | `#6cc7d4` | Streaming pulse dot; `● live` statusbar indicator color |
| Accent — Tool success | `--accent-green` | `#6fd28f` | Tool state `ok` glyph; `liveLabel` 'live' color; stream settled text |
| Accent — Permission / gate | `--accent-red` | `#e87b7b` | Permission gate border tint; Deny button color; spawn-failure error text |

| Tool glyph | `--tool` | `#c084d4` | Tool event glyph in `pending` state — role-semantic token from `variant-b.css` |
| Foreground 0 | `--fg-0` | `#e8eaf0` | Task text; stream.delta growing text; final text |
| Foreground 1 | `--fg-1` | `#aab0c0` | Tool line collapsed label; session list item text |
| Foreground 2 | `--fg-2` | `#6a7080` | Tool args summary (collapsed); plan prose; generic fallback row type label |
| Foreground 3 | `--fg-3` | `#444a5a` | System events (server.connected, session.idle); dim rows; elapsed counter |
| Destructive | `--accent-red` | `#e87b7b` | Deny button + gate border — permission gates only |

**Accent reserved for:**
- `--focus` (`#5a7cff`): Attach and Retry start interactive buttons; keyboard focus rings
- `--accent-cyan` (`#6cc7d4`): Streaming pulse dot; statusbar live indicator dot
- `--accent-green` (`#6fd28f`): Tool `ok` state glyph; liveLabel 'live' text; stream settled paragraph
- `--accent-red` (`#e87b7b`): Permission gate border/background tint; Deny button; spawn-failure error; ended-state dot (err tier from ExitBanner)
- `--accent-amber` (`#e8b86c`): Warning and probable event rows; permission gate `dimension="confidence"` or `dimension="budget"` label
- `--tool` (`#c084d4`): Tool event glyph (pending state) only

**Honest-data discipline (V14 rule — carried forward):**
- `--accent-cyan` streaming pulse animates ONLY while `stream.delta` events are actively arriving (SSE connected and turn in flight). Remove the animation class on `stream.finalize`, `session.idle`, stream disconnect, or server death.
- `● live` statusbar indicator uses `--accent-cyan` ONLY when a real server session backs the selected pane (liveLabel = 'live'). When liveLabel = 'snapshot', apply `--fg-3` (dim, no dot animation).

## Component Inventory

**Focal-point declaration:** the task header row is the transcript's primary
anchor at full contrast (`--fg-0`, weight 600); every downstream row descends
the foreground scale `--fg-1` → `--fg-2` → `--fg-3` so the eye lands on the
task first and reads activity in decreasing emphasis.

### 1. Structured Protocol Pane Body (`<ProtocolPane>`)

Renders inside `.pane-body` (existing chrome unchanged). Replaces xterm for
RunCommandBar-native sessions. PTY panes are untouched.

**Container:**
```
.protocol-pane
  overflow-y: auto
  height: 100%
  padding: 8px 12px
  background: var(--bg-0)
  font-family: var(--font-mono)
  font-size: 11px
  line-height: 1.62
  scrollbar-color: var(--bg-3) transparent
  user-select: text   ← text in transcript is selectable (override global none)
```

Sticky-bottom auto-scroll unless the user scrolled up (D-09). Keyboard
scrollable via arrow keys and Page Up/Down when the pane has focus (VLIVE-04
a11y requirement).

#### 2g. Generic Fallback Row (all other union members)

Covers: `server.connected`, `session.idle`, `banner`, `status`,
`cognition_loaded`, `cognition_overflow`, `principles_overflow`, `clarify`,
`warning`, `probable`, `budget.updated`, `confidence.updated`, `gate.updated`.

Nothing is silently dropped. Every event produces at least this row.

```
.proto-generic-row
  display: flex
  align-items: center
  gap: 8px
  padding: 4px 0
  color: var(--fg-3)
  font-size: 10px
```

### 6. Ended Banner / Dim State (D-11)

When the server dies mid-session, the transcript appends an ended-state row
reusing ExitBanner visual language, and the pane chrome dims.

**Ended banner row (appended to transcript scroll container):**

Reuse `<ExitBanner>` from `apps/voss-app/src/pane/ExitBanner.tsx` with:
- `exitCode`: use `0` for clean `session.idle` / `1` for server death
- `onRestart`: no-op (no Restart button for server-death case; the next native
  run spawns a fresh server per D-11 / VLIVE-07)

The pane banner is the **transcript** ended row (appended inline, not
absolutely positioned), so it scrolls with the transcript. This differs from
PTY panes where ExitBanner is position:absolute at the bottom.

```
.proto-ended-row
  (same markup as .exit-banner but rendered inline in transcript flow)
  margin-top: 8px
  .eb-msg content: "[session ended]"
  .eb-dot class: "err" tier → color: var(--accent-red)
  (no Restart button — server death; next run spawns fresh)
```

**Pane chrome dim:** when ended, apply to `.pane`:
```
.pane--proto-ended
  opacity: 0.75
  transition: opacity 200ms ease
```

Reduced-motion: skip the opacity transition.
```
html.reduced-motion .pane--proto-ended,
@media (prefers-reduced-motion: reduce) { .pane--proto-ended }
  transition: none
```

## Animations

| Name | Keyframes | Duration | Applied To |

## Interaction Contracts

### Transcript Scrolling
- Auto-scroll to bottom on new events (sticky-bottom behavior).
- Auto-scroll OFF when the user scrolls up (detect scroll position delta > 20px above bottom).
- Auto-scroll resumes when the user scrolls back to within 20px of the bottom.
- Keyboard: arrow keys, Page Up/Down scroll the transcript when the pane has focus.
- Tab does NOT focus the transcript container — it traverses to the next focusable element (permission gate buttons if present).

### Tool Line Expansion (D-07)
- Collapsed state is the default for every tool line.
- Click anywhere on the row toggles expanded/collapsed.
- Keyboard: the collapsed row is NOT in the Tab order (it is not interactive enough to warrant it); the expanded state reveals no actionable controls.
- No animation on expand/collapse (Variant B: transitions: none pattern).

### Permission Gate
- Tab order within the gate: Deny → Allow once → Allow for scope.
- Pressing Enter on a focused button triggers the choice.
- After a choice is submitted, all three buttons are immediately disabled (`.proto-pgbtn:disabled`). The gate resolves to the resolved display once the POST completes (or errors — on error, buttons re-enable).
- One reply clears BOTH the inline gate AND the AttentionQueue row for the same permission id (VLIVE-05).

### Attach Action
- Clicking "Attach" on a session-list row immediately triggers the D-02 grid-insertion path (same as sidebar quick-launch).
- No confirmation dialog — attach is not destructive.
- If the server is not running (cold), show the boot placeholder in the new pane while the sidecar respawns.

## D-07 Mockup Deviation — Formal Record

**Decision source:** CONTEXT.md D-07 (locked user decision).

**What the mockup shows:** Tool lines display inline code excerpts (e.g. the
`fs_edit` diff in the executor-1 pane) always visible in the transcript.

**What V15 builds instead:** All tool lines are collapsed one-liners by
default. The summary field from the `tool` event provides the one-line label
(`fs_edit src/auth/passkey.ts +34 −2`). Clicking the row expands to show
full args, result, and any excerpt.

**Rationale (D-07):** Keeps transcript density readable; reduces visual noise
for long multi-tool runs; mirrors the collapsed-by-default pattern common in
modern AI coding tool UIs.

**Executor note:** Do not render excerpts or diffs in the collapsed state. The
`summary` field from the event wire is the collapsed text.
