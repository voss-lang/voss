## Token System Architecture

**Single source of truth:** CSS custom properties defined in `apps/voss-app/src/styles/variant-b.css`.
Tailwind v4 `@theme inline { }` in `apps/voss-app/src/index.css` maps every token to a Tailwind utility.
Components use only Tailwind utility classes — never raw hex or px values in component code.
Runtime theme swap: `document.documentElement.style.setProperty("--token", value)` at mount, sourced from `invoke("get_theme_overrides")`.

Source: CONTEXT D-01, D-02, D-08, D-09; RESEARCH Pattern 1.

## Spacing Scale

Variant B is a dense terminal aesthetic — spacing deviates from standard 8-point web scale in the small end. All values are multiples of 2px (terminal-grid convention) and multiples of 4px at lg and above.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-glyph gap, inline separator padding |
| sm | 6px | Traffic-light button gap (6px between circles per sketch) |
| md | 8px | Compact element spacing, scrollbar width |
| base | 10px | Titlebar horizontal padding, cell header horizontal padding |
| lg | 12px | Titlebar left offset for traffic-light cluster (`padding-left: 12px`) |
| xl | 16px | Gap between titlebar regions |
| 2xl | 24px | Future section-level spacing (not used in A1 chrome) |

**Exceptions and hard measurements:**
- Titlebar height: **22px** (Variant B `.b-head { height: 22px }`) — locked, not derived from scale.
- Traffic-light circles: **12px diameter, 6px gap** — locked from sketch `.traffic span { width: 12px; height: 12px }`, `.traffic { gap: 6px }`.
- Traffic-light cluster left offset: **12px** from window left edge.
- Scrollbar width: **8px** (confirmed: `themes/default.css ::-webkit-scrollbar { width: 8px }`).
- Border: **1px** everywhere — no exceptions in A1.
- Border radius: **0px** everywhere — Variant B locked. No rounding on any element in A1 (the preset-switcher in the sketch uses `var(--radius-md)` but that is sketch convenience CSS; A1 titlebar preset switcher uses 0 radius per D-02 Variant B spec).

Source: sketch `.b-head`, `.traffic`, `themes/default.css`, CONTEXT D-02.

## CSS Variable to Tailwind Mapping Contract

Define in `apps/voss-app/src/index.css` after `@import "tailwindcss"` and `@import "./styles/variant-b.css"`:

```css
@theme inline {
  /* Backgrounds */
  --color-bg-0:        var(--bg-0);
  --color-bg-1:        var(--bg-1);
  --color-bg-2:        var(--bg-2);
  --color-bg-3:        var(--bg-3);
  /* Borders */
  --color-border:      var(--border);
  --color-border-bright: var(--border-bright);
  /* Focus */
  --color-focus:       var(--focus);
  /* Foreground */
  --color-fg-0:        var(--fg-0);
  --color-fg-1:        var(--fg-1);
  --color-fg-2:        var(--fg-2);
  --color-fg-3:        var(--fg-3);
  /* Accents */
  --color-accent-green:   var(--accent-green);
  --color-accent-amber:   var(--accent-amber);
  --color-accent-red:     var(--accent-red);
  --color-accent-cyan:    var(--accent-cyan);
  --color-accent-magenta: var(--accent-magenta);
  --color-accent-blue:    var(--accent-blue);
  /* Role semantic */
  --color-user-msg:    var(--user-msg);
  --color-tool:        var(--tool);
  --color-reviewer:    var(--reviewer);
  /* Fonts */
  --font-mono:         var(--font-mono);
  --font-ui:           var(--font-ui);
}
```

This generates utilities: `bg-bg-0`, `text-fg-0`, `border-border`, `text-accent-green`, `font-mono`, etc.
Components MUST use these utility classes. They MUST NOT use inline `style="color: #..."` except in the `applyThemeOverrides` function itself.

Source: RESEARCH Pattern 1, CONTEXT D-01.

All three regions are children of a single `<div class="titlebar">` with `display: flex; align-items: center`:

## Theme Override System Contract

### Rust Side (D-09)

Tauri command `get_theme_overrides` reads `~/.config/voss-app/settings.json` (note: use `dirs::home_dir().join(".config")` NOT `dirs::config_dir()` — macOS `config_dir()` returns `~/Library/Application Support`, not `~/.config`). Returns `HashMap<String,String>` where keys are CSS variable names (e.g. `"--bg-0"`) and values are CSS values (e.g. `"#1a1a2e"`). Absent file → empty map → pure Variant B. Malformed JSON → empty map (log error, do not crash).

### Solid Side (D-08)

Call `invoke("get_theme_overrides")` in `onMount` of the root App component. Iterate entries and call `document.documentElement.style.setProperty(key, value)` for each. Applied before first meaningful paint cycle completes (acceptable in A1; theme flash is invisible when settings file is absent, which is the primary dev path).

### Overridable Tokens

Any CSS variable in `variant-b.css` may be overridden by the settings file. The settings JSON may only contain a `theme` object — any other keys are ignored. Example valid `settings.json`:
```json
{
  "theme": {
    "--bg-0": "#1a1a2e",
    "--accent-green": "#50fa7b"
  }
}
```

Source: CONTEXT D-08, D-09; RESEARCH Pattern 3, Pitfall 8.

## Anti-Patterns (enforcement notes for executor)

These must NOT appear in A1 implementation:

| Anti-Pattern | Why Forbidden | Source |
|--------------|---------------|--------|
| `tailwind.config.js` theme.extend | Tailwind v4 uses `@theme inline` in CSS — no JS config | RESEARCH state-of-the-art |
