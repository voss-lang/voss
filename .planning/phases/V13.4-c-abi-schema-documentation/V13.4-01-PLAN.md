<!-- Extracted from .planning/PROTOCOL.md so the executor authors accurately WITHOUT re-deriving. The doc SUMMARIZES + POINTS AT these; it does NOT copy the schema body. -->

Transport (PROTOCOL §1, §2):
- REST (commands, request/response) + SSE (server->client stream). NOT WebSocket.
- Base URL: http://127.0.0.1:<port> — loopback only, no TLS on loopback.
- `voss serve` binds 127.0.0.1:0 then prints exactly ONE line of JSON to stdout: {"v":1,"port":54123,"token":"<url-safe-32-byte>"}.
- Every JSON body and every SSE data payload carries "v": 1. Breaking change increments v + ships a migration note.
- Content types: REST = application/json; stream = text/event-stream.
- OpenAPI 3.1 at GET /openapi.json (FastAPI). SSE union documented via _EventEnvelope schema component.

Auth (PROTOCOL §2):
- Every request (REST + the SSE GET) must carry Authorization: Bearer <token>. 401 at middleware (constant-time compare); no token never reaches route logic.
- Token is per-server-process, ephemeral, NEVER persisted.

Endpoints (PROTOCOL §4): POST/GET /session, GET/DELETE /session/:id, POST /session/:id/message (202 accepted, async), GET /session/:id/events (SSE), POST /session/:id/abort, POST /session/:id/permission, GET /doctor, GET /openapi.json. Voss-native additive: POST /session/:id/budget, GET /session/:id/confidence. One running turn per session; POST /message during a turn -> 409.

SSE event model (PROTOCOL §6): wire framing `event: <type>` / `data: {"v":1,...}` / `id: <seq>`. First event always server.connected; turn completion signalled by session.idle. The event union is a pydantic discriminated union Field(discriminator="type") wrapped as {event: AgentEvent}, forced into OpenAPI components. The `type` literal is BOTH the SSE event: name AND the serde tag.

Parts / message schema (PROTOCOL §5): a turn = messages composed of parts, shared envelope {type, ...}. ToolState discriminated on status (pending/running/completed/error).

Permission (§7), abort (§8), error model (§9): permission.updated {id,tool_name,args,dimension} + POST /permission {v,id,choice}; abort -> stream.delta("[interrupted]") + finalize; REST errors = HTTP status + {v:1, detail}.

CRITICAL — the union member COUNT is NOT to be hardcoded as the contract. PROTOCOL §6 lists the events, but the AUTHORITATIVE member source the doc names is contracts/events.schema.json (per D-03). The doc states "member set is authoritative-from-schema", documents the `type` discriminator + `v` version + required/optional + scalar/composite reading guidance — it does NOT freeze an enumerated list as the contract.
