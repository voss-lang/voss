# Phase V13.4 — Validation Strategy

> Per-phase validation contract. **Doc-only phase** — the acceptance checks ARE the automated verifications: grep assertions on the prose doc, a references-resolve script, a no-fork grep, a no-code grep, and a `PROTOCOL.md` zero-diff check. No test framework / no simulator.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | shell assertions (`grep`, `test`, `git diff`) — no test runner |
| **Config file** | none — checks are inline shell/CI assertions |
| **Quick run command** | `grep -q . docs/native-embedding.md` (doc exists + non-empty) |
| **Full suite command** | run the VSDK-C-06 references-resolve check + the no-fork / no-code / PROTOCOL-zero-diff greps (planner wires these) |
| **Estimated runtime** | < 2s (pure text checks) |

> The references-resolve check (VSDK-C-06) is the single hard gate. It `t.Skip`-equivalent (warns, exits 0) while V13.1's `contracts/` + V13's tier taxonomy are unshipped, and fails (exits non-zero) on any missing cited path once it is expected to pass post-V13.1/V13.

---

## Sampling Rate

- **After every task commit:** re-grep the doc for the section just authored.
- **Before `/gsd-verify-work`:** all acceptance greps green; `git diff -- .planning/PROTOCOL.md` empty; no-code grep clean; refs-resolve check run (pass post-V13.1/V13, or warn-skip with the unshipped-upstream note).
- **Max feedback latency:** < 2s.
