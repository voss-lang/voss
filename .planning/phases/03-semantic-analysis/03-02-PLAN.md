<objective>
Implement ANLY-01: a narrow semantic check that warns when `probable<T>` is used where plain `T` is expected without an explicit confidence gate, with diagnostic spans pointing to the unsafe expression.
</objective>

<threat_model>
Threats:
- T-03-02-A: Analyzer trusts `.value` access as explicit consent and allows unsafe low-confidence values into business logic.
- T-03-02-B: Branch-local narrowing leaks after an `if`, suppressing warnings on paths that were never gated.
- T-03-02-C: Unknown types produce noisy false positives that hide real warnings.
- T-03-02-D: Function-call analysis executes or imports user functions to discover signatures.

Mitigations:
- Emit `ANLY001` for `.value` on `probable<T>` unless the root identifier is in `Scope.gated`.
- Copy scopes for `then_body` and `else_body`; never mutate the parent gated set when entering a branch.
- `UnknownType` is non-warning and non-assignable only by omission; it never triggers `ANLY001`.
- Predeclare signatures by reading `FnDecl`/`AgentDecl` AST nodes only; no import resolution or function execution.
</threat_model>

Extend `voss/analyzer.py` with the minimal internal type model and shallow inference.

Add frozen private dataclasses/classes:
```python
class VossType: ...
@dataclass(frozen=True, slots=True)
class PrimitiveType(VossType): name: str
@dataclass(frozen=True, slots=True)
class NamedType(VossType): name: str
@dataclass(frozen=True, slots=True)
class ListType(VossType): item: VossType
@dataclass(frozen=True, slots=True)
class DictType(VossType): key: VossType; value: VossType
@dataclass(frozen=True, slots=True)
class ProbableType(VossType): inner: VossType
class UnknownType(VossType): ...
UNKNOWN = UnknownType()
```

Add helpers:
- `_type_from_type_expr(type_expr: TypeExpr | None) -> VossType`
- `_type_display(type_: VossType) -> str`
- `_types_equal(left: VossType, right: VossType) -> bool`
- `_root_identifier(expr: Expr) -> str | None`
- `_is_gated(expr: Expr) -> bool`
- `_warn_unguarded_probable(actual: ProbableType, expected: VossType, expr: Expr) -> None`
- `_check_expected(expr: Expr, expected: VossType) -> VossType`
- `_infer_expr(expr: Expr) -> VossType`

Normalization rules:
- `TypeRef(name=QualName(("probable",)), generics=(T,))` -> `ProbableType(T)`.
- `string`, `int`, `float`, `bool`, `null` -> `PrimitiveType(...)`.
- `list<T>` -> `ListType(T)`.
- `dict<K,V>` -> `DictType(K,V)`.
- other qualified names -> `NamedType(".".join(parts))`.
- Missing/unknown annotations -> `UNKNOWN`.

Scope/signature rules:
- `Scope.symbols: dict[str, VossType]`.
- Add private `FunctionSignature(params: dict[str, VossType], return_type: VossType)`.
- Before walking top-level statements, predeclare every `FnDecl` and `AgentDecl` with known parameter and return annotations.
- For `LetStmt` with `type_annot`, call `_check_expected(value, annotated_type)` before binding the variable to the annotated type.
- For `LetStmt` without annotation, bind to `_infer_expr(value)`.
- For `ReturnStmt`, if `self._return_type` is not `UNKNOWN`, call `_check_expected(value, self._return_type)`.
- For `Call` where callee is `Identifier` and signature is known, check each positional/named arg against the matching parameter type.

Confidence gate rules:
- `ConfidenceGate(target=Identifier("intent"), op=">=" or ">", threshold=...)` narrows `intent` only in the `then_body`.
- For `<` or `<=`, v1 may narrow only the else body, but tests only require that `>=` then-body narrows and else does not.
- No narrowing after the `if` statement.
- `Member(obj=Identifier("intent"), attr="value")` on `ProbableType(T)` returns `T`; if `intent` is not gated in current scope, emit `ANLY001` at the member span.

Diagnostic target:
```python
Diagnostic(
    severity="warning",
    code="ANLY001",
    message=f"unguarded {actual_display} used where {expected_display} is expected",
    span=expr.span,
    hint="Add a confidence gate such as if intent @ p >= 0.80 { ... } or pass intent.value only inside the gated branch.",
)
```
Only emit when expected is not `UNKNOWN`, actual is `ProbableType(inner)`, `inner == expected`, and expression is not gated.
