<objective>
Implement ANLY-03: collect all `similar("...")` match cases during semantic analysis, precompute case embeddings through a fakeable builder seam, and write a deterministic program-level manifest to `.voss-cache/<program>.idx`.
</objective>

<threat_model>
Threats:
- T-03-04-A: A malicious or mistaken `cache_dir` escapes the project/cache boundary and overwrites arbitrary files.
- T-03-04-B: Default tests instantiate sentence-transformers and become slow, flaky, or network-dependent.
- T-03-04-C: Multiple match blocks overwrite each other or collide in a single flat runtime index.
- T-03-04-D: Case labels are derived from source/body text and become unstable or leak source content.

Mitigations:
- Resolve `cache_dir` and target path, ensure target path is under resolved `cache_dir`, and emit `ANLY003` error if not.
- Accept an injectable `IndexBuilder`; tests pass a fake builder with deterministic embeddings.
- Always write one manifest with `matches: [...]`, one entry per match block.
- Use stable `match_<line_start>_<col_start>` IDs and ordinal `case_0`, `case_1` labels.
</threat_model>

Add match-index emission to `voss/analyzer.py`.

Default builder:
```python
class SemanticMatcherIndexBuilder:
    model = DEFAULT_LOCAL_MODEL
    def __init__(self, *, model: str = DEFAULT_LOCAL_MODEL) -> None: ...
    def build_cases(self, cases: Sequence[tuple[str, str]]) -> list[dict[str, object]]:
        matcher = SemanticMatcher(cases, threshold=0.75, model=self.model)
        return matcher.to_index()["cases"]
```
Import `DEFAULT_LOCAL_MODEL` and `SemanticMatcher` from `voss_runtime.semantic` only in this builder section, not in token-estimation code.

Analyzer collection:
- Add `self._match_entries: list[dict[str, object]]`.
- Add `_visit_match_stmt(match: MatchStmt) -> None`.
- Collect only cases where `case.pattern` is `SimilarPattern`.
- Build pairs in runtime order: `(description, label)` where `description = pattern.text`, `label = f"case_{ordinal}"`.
- Use `match_id = f"match_{match.span.line_start}_{match.span.col_start}"`.
- Use threshold `match.threshold if match.threshold is not None else 0.75`.
- Append:
```python
{
    "match_id": match_id,
    "threshold": threshold,
    "cases": built_cases,
}
```
- Still visit each case body so ANLY-01/ANLY-02 checks inside match arms continue to run.

Program index write:
- At the end of `analyze_program`, if `emit_indexes` and entries exist, write one file.
- Program stem: `Path(source_path).stem` when source_path is provided; otherwise `"program"`.
- Manifest:
```python
{
    "version": 1,
    "program": program_stem,
    "model": self.index_builder.model,
    "matches": self._match_entries,
}
```
- Target: `Path(cache_dir) / f"{program_stem}.idx"`.
- Create `cache_dir` if needed.
- Validate `target.resolve().is_relative_to(cache_dir.resolve())`; if validation fails, append error diagnostic:
```python
Diagnostic(
    severity="error",
    code="ANLY003",
    message="refusing to write embedding index outside cache directory",
    span=program.span,
    hint="Use a cache_dir inside the project, such as .voss-cache.",
)
```
- Write atomically: write JSON to `target.with_suffix(target.suffix + ".tmp")`, then `replace(target)`.
- Append `EmittedIndex(match_id="program", path=target, case_count=sum(...), threshold=0.75, model=self.index_builder.model)` or, preferably, one `EmittedIndex` per match entry with the shared path and each match threshold.

JSON formatting: `json.dumps(manifest, indent=2, sort_keys=False)`.
