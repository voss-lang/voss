<objective>
Implement ANLY-02: a conservative static token-budget estimator for `ctx` blocks that warns when obvious literal and included context exceeds the declared token budget.
</objective>

<threat_model>
Threats:
- T-03-03-A: Static analysis calls provider token counters or external model APIs, making `voss check` non-hermetic.
- T-03-03-B: Over-aggressive unknown-value estimates produce noisy warnings and train users to ignore analyzer output.
- T-03-03-C: A ctx budget with a non-token unit is misread as tokens and produces misleading diagnostics.
- T-03-03-D: Large source literals in diagnostics leak excessive user data into logs.

Mitigations:
- Implement `DefaultTokenEstimator` with local heuristics only; tests grep source for absence of provider/API imports.
- Unknown calls and unresolved identifiers contribute no warning-producing estimate.
- Only inspect `BudgetArg.name == "budget"` and `BudgetArg.unit == "tokens"` for `CtxBlock`.
- Diagnostic message reports only numeric estimate and budget, not the literal contents.
</threat_model>

  <action>
Add a deterministic `DefaultTokenEstimator` to `voss/analyzer.py` and wire it into `Analyzer`.

Target class:
```python
class DefaultTokenEstimator:
    def estimate_expr(self, expr: Expr, scope: Scope) -> int | None: ...
    def estimate_stmt(self, stmt: Stmt, scope: Scope) -> int | None: ...
```

Heuristics:
- `StringLit`: `max(1, len(expr.value) // 4)`.
- `IntLit`, `FloatLit`, `BoolLit`, `NullLit`: `1`.
- `Identifier`: `scope.estimates.get(expr.name)`.
- `BinOp(op="+")`: sum left and right when both are known; otherwise `None`.
- `ListLit`: sum known items plus item count overhead; unknown item makes expression `None`.
- `DictLit`: sum known keys/values plus pair count overhead; unknown makes expression `None`.
- `Call`: if callee is `Identifier("ask")` and first positional arg is statically known, return that estimate; all other calls return `None`.
- `Member`, `Index`, `Lambda`, `SpawnExpr`, unknown expression nodes: `None`.
- `IncludeStmt`: estimate `value`.
- `YieldStmt` containing `Call(Identifier("ask"), ...)`: estimate the ask prompt.
- Other statements: `0` or `None` only when dynamic; they must not force a warning.

Analyzer wiring:
- In `Analyzer.__init__`, set `self.token_estimator = token_estimator or DefaultTokenEstimator()`.
- When visiting a `LetStmt`, if initializer estimate is not `None`, store `scope.estimates[name] = estimate`.
- Add helper `_ctx_token_budget(ctx: CtxBlock) -> int | None` that returns budget only when `ctx.budget.name == "budget"` and `ctx.budget.unit == "tokens"`.
- Add `_estimate_ctx_body(ctx: CtxBlock) -> int`, summing only known estimates from `IncludeStmt`, literal `ask(...)` prompts, and statically estimable statements. Unknown statements contribute 0.
- Add `_visit_ctx_block(ctx: CtxBlock) -> None`; emit:
```python
Diagnostic(
    severity="warning",
    code="ANLY002",
    message=f"ctx block static token estimate {estimate:,} exceeds declared budget {budget:,}",
    span=ctx.span,
    hint="Increase the ctx budget, include less context, or add a more aggressive compression strategy.",
)
```
Then visit the ctx body normally so nested probable checks still run.

Security source check: do not add imports or references to `ModelProvider`, `litellm`, `openai`, `anthropic`, `sentence_transformers`, or `chromadb` in `voss/analyzer.py`.
